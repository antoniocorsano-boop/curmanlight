## Passo 3 — File creato

`tools/json-to-mappa-dati-adapter.mjs`

### Funzioni implementate

| Funzione                            | Logica                                                                                               |
| ----------------------------------- | ---------------------------------------------------------------------------------------------------- |
| `readJson(path)`                    | Legge e parse JSON; errore se file non trovato o JSON invalido                                       |
| `validateInput(data)`               | Controlla campi obbligatori; warning su stderr per dati mancanti                                     |
| `buildStruttureSostanziali(units)`  | Raggruppa per `ambito`; ordina alfabeticamente                                                       |
| `buildNodiDisciplinari(units)`      | Raggruppa per `nucleo` (o `nucleoFondante`); ordina alfabeticamente                                  |
| `buildProgressioneVerticale(units)` | Raggruppa per `ordine`+`classe`/`fascia`; ordina Infanzia→Primaria→Secondaria, poi per classe/fascia |
| `buildDecisioniCurricolari(data)`   | Pass-through se presenti; array vuoto altrimenti                                                     |
| `toMappaDati(data)`                 | Compone l'oggetto MAPPA_DATI completo                                                                |

### Preflight warnings (stderr)

- `ambito` mancante o vuoto
- `nucleo` mancante o vuoto
- unità Infanzia senza `fascia`
- unità Primaria/Secondaria senza `classe`

### Errori bloccanti (stderr + exit 1)

- File non trovato
- JSON invalido
- `unitaApprendimento` non array o vuoto
- `disciplina` mancante

## Limiti noti

1. `struttureSostanziali[].nome` = `ambito` — identico. Le MAPPA_DATI hardcoded hanno `nome` leggermente accorciato.
2. `progressioneVerticale[].unita` = riferimenti sintetici. La MAPPA_DATI hardcoded ha `descrizioneProgressione` testuale.
3. `nodiDisciplinari[].descrizione` non generata (assente nel contratto CML-141). La MAPPA_DATI hardcoded l'ha.
4. Output non 1:1 con MAPPA_DATI hardcoded — voluto: il tool produce una struttura più ricca con riferimenti unità, che potrà essere raffinata.
