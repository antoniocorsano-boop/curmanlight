## Diagnosi problema shape/BOM

Diagnosi eseguita:

1. `node tools/validate-cml-normalized-curriculum.mjs`:
   - 10 file
   - 118 unita
   - `overallValid: true`
   - 0 errori
2. `node tools/test-runtime-mappa-dati-shape.mjs` prima della modifica:
   - PASS 9/9
   - Musica non inclusa nel set `FILES`
3. Verifica runtime Musica:
   - `MUSICA_MAPPA_DATI` presente
   - branch `musica` presente
   - pulsante/entry Musica presente
4. Verifica dati Musica:
   - adapter e transformer producono JSON parsabile
   - `content/curriculum/musica.normalized.json` non modificato
5. Verifica encoding:
   - nessun BOM UTF-8 nei file `.normalized.json`
   - nessun byte nullo nei file `.normalized.json`

Esito: il problema riproducibile non e' nei dati e non e' nel runtime. Il test era da riallineare a 10 discipline. E' stata aggiunta anche una guardia minima nel parsing del test per rimuovere un eventuale BOM UTF-8 iniziale dall'output JSON e segnalare byte nulli come possibile UTF-16/encoding non valido.

## Intervento effettuato su test shape

Modificato solo `tools/test-runtime-mappa-dati-shape.mjs`:

- aggiunto `musica.normalized.json` alla lista `FILES`;
- aggiunta funzione `normalizeJsonText(raw, label)`;
- parsing JSON aggiornato da `JSON.parse(raw.trim())` a `JSON.parse(normalizeJsonText(raw, filepath))`.

La normalizzazione non modifica runtime o dati curricolari. Rimuove solo un eventuale BOM iniziale nell'output JSON del test e non ignora byte nulli: li segnala con errore esplicito.
