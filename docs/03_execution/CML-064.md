# CML-064 — TECHNOLOGY_PACK_RUNTIME_READONLY_PREVIEW

## Obiettivo
Integrare nel runtime CurManLight una prima anteprima consultiva, non modificabile, del pacchetto disciplinare normalizzato di Tecnologia (CML-062), senza alterare export, import, schema `.cml`, salvataggio locale o workflow GitHub Pages.

## Rapporto con CML-061/CML-062
- CML-061: contratto dati normalizzato (`cml-normalized-curriculum-v1`).
- CML-062: pacchetto Tecnologia (13 unità, validazione `valid: true`).
- CML-064: integrazione read-only nel runtime esistente.

## Scelta UX
Aggiunta di una sezione `tecnologia-norm` all'interno del tab "Curricolo di istituto" (`#tab-curricolo`), posizionata **prima** del viewer esistente (`#curricolo-body`). Questo permette:
- una schermata, una decisione;
- quadro sintetico (statistiche + badge) prima del dettaglio;
- dettaglio progressivo espandibile per unità;
- nessun nuovo pulsante globale, nessuna nuova azione;
- nessuna illusione di approvazione automatica.

## Cosa è stato aggiunto
- Container `#tecnologia-norm-panel` in `index.html` (linea 1096).
- Costante JavaScript `TECNOLOGIA_NORM` con tutti i dati normalizzati (17KB).
- Funzione `renderTecnologiaNorm()` per rendering read-only.
- Integrazione in `setTab('curricolo')` per chiamare anche `renderTecnologiaNorm()`.
- Voce "Tecnologia (pacchetto norm.)" nel menu mobile overlay.
