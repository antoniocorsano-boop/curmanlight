# CML-500 — Department Decision

## Obiettivo

Consentire al ruolo Dipartimento di registrare nel browser una decisione per ogni proposta presente nella coda locale introdotta con CML-499.

## Implementazione

- quattro esiti disponibili: accettata, respinta, modificata, rinviata;
- persistenza locale della decisione nella stessa coda versionata;
- nota dipartimentale facoltativa;
- testo concordato obbligatorio per l’esito `modificata`;
- modifica o rimozione della decisione già registrata;
- riepilogo delle proposte da decidere e già decise;
- migrazione compatibile delle righe CML-499 prive del campo `decision`;
- nessun backend, upload, telemetria o autenticazione;
- nessuna esportazione `department_outcome`, riservata a CML-501.

## Criteri di accettazione

1. Ogni riga della coda può ricevere uno dei quattro esiti previsti.
2. Una decisione `modificata` non viene salvata senza testo concordato.
3. Le decisioni restano disponibili dopo il reload del browser.
4. Una decisione può essere modificata o rimossa.
5. Le proposte importate prima di CML-500 vengono lette con `decision: null`.
6. La UI non genera ancora file `.cml` di esito.
