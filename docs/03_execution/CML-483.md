# CML-483 — Percorso guidato di valutazione per docenti non tecnici

## Stato

`LOCAL_INTERACTIVE_VALIDATION_COMPLETED_READY_FOR_REMOTE_REVIEW`

## Obiettivo

Introdurre in CurManLight un percorso guidato, locale e reversibile, destinato a docenti non tecnici, che consenta di:

- utilizzare alcune funzioni significative dello strumento;
- riflettere sull'esperienza d'uso;
- annotare osservazioni facoltative;
- esportare localmente una restituzione leggibile;
- non trasmettere dati all'esterno.

## Branch

`codex/cml-483-guided-teacher-evaluation`

Issue GitHub: #73

## Decisione sull'applicazione interessata

L'analisi della baseline successiva a CML-461/CML-462 e delle slice CML-463–CML-482 mostra due superfici distinte:

- applicazione legacy: `index.html` e `_published_snapshot/netlify-current/index.html`, mantenute sincronizzate fra loro;
- candidato operativo React: `curman-react/`, su cui sono stati realizzati B02, B03, applicabilità temporale e registro delle versioni.

Il percorso di valutazione è stato quindi introdotto **solo nel candidato React**, per osservare le funzioni attive oggetto delle più recenti verifiche. La coppia legacy resta invariata e sincronizzata; non deve ricevere una duplicazione automatica della funzione senza una decisione di prodotto separata.

## Implementazione

File funzionali:

- `curman-react/src/components/evaluation/GuidedTeacherEvaluation.tsx`;
- `curman-react/src/views/HomeView.tsx`.

La Home presenta la sezione:

**Esplora lo strumento e lascia le tue osservazioni**

Il percorso comprende cinque tappe:

1. Orientarsi nella Home.
2. Consultare una disciplina.
3. Osservare un'attività di aggiornamento senza obbligo di registrare decisioni.
4. Valutare guida e supporti contestuali.
5. Restituire punti di forza, difficoltà e miglioramento prioritario.

## Comportamenti

- note facoltative per ogni tappa;
- avanzamento, salto e ritorno alla tappa precedente;
- chiusura del pannello e ripresa dalla Home;
- persistenza in `localStorage` con chiave versionata `curmanlight:guided-teacher-evaluation:v1`;
- identificativo locale generato sul dispositivo;
- nessun dato personale obbligatorio;
- disponibilità facoltativa a un secondo test;
- esportazione JSON locale con schema `cml-guided-teacher-evaluation-v1`;
- cancellazione esplicita del percorso locale;
- nessuna scrittura nei dati curricolari, nei Gap package o nelle decisioni B03;
- nessuna richiesta di rete introdotta dal percorso.

## Privacy e reversibilità

Il componente utilizza esclusivamente API del browser:

- `localStorage` per la ripresa;
- `Blob` e `URL.createObjectURL()` per l'esportazione;
- nessun `fetch`, servizio remoto, autenticazione o telemetria.

La funzione è isolata in un componente dedicato e può essere rimossa eliminando il componente e il relativo inserimento in `HomeView`.
