## Issue classificati per severità

### P1 — Alto impatto utente

1. **Tabbar bottoni a 400px viewport**: `font-size:9px; padding:6px 3px; min-width:0` — target touch insufficiente per utenti con dita grandi o motricità ridotta.

### P2 — Medio (chiarezza/polish)

2. **Mancanza `aria-current` su tabbar**: utenti screen reader non scono qual è il tab attivo.
3. **Mancanza `aria-pressed` su mappa-disc-btn**: utenti screen reader non scono quale disciplina è selezionata.
4. **Mancanza `aria-current` su subnav bottoni**: stessi problema dei tab.
5. **Textarea preview senza label**: `#disc-md-preview` manca `<label for="disc-md-preview">`.
6. **`.motto-link:focus{outline:none}`**: indicatore di focus rimosso su link footer.
7. **Nessuna media query sotto 380px**: viewport 360px può causare overflow minore.

### P3 — Bassa priorità

8. Mappa disc pulsanti inline style dimensione fissa (~34px) — al limite WCAG.
9. Icone emoji come solo contenuto in alcuni bottoni filtro.

## Raccomandazione per CML-230

**Microfix mirato**: Correggere P1 + P2 più impattanti con modifiche minime:

1. **Tabbar 400px touch target** (1 riga CSS): rimuovere `min-width:0`, aumentare `padding` a minimo 8px
2. **`aria-current` su tabbar** (2 righe JS in `setTab()`)
3. **`aria-pressed` su mappa-disc-btn** (2 righe JS in `setMappaDisciplina()`)
4. **`aria-current` su subnav bottoni** (1 riga JS in `setTab()`)
5. **Label per textarea preview** (1 riga HTML)
6. **Focus outline su motto-link** (1 riga CSS)

Alternative rigettate:

- Media query per 360px: utile ma non giustifica modifica runtime singola
- Icone emoji: risoluzione complessa, impatto basso
