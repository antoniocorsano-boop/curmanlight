# CML-502 — Referent Outcome Import

## Obiettivo

Consentire al ruolo Referente di importare localmente uno o più file `.cml` di tipo `department_outcome`, validarli, individuare duplicati nel gruppo selezionato e consultarne gli esiti in modalità sola lettura.

## Implementazione

- parser dedicato `departmentOutcomeImport.ts`;
- validazione di schema v1.0, ruolo, disciplina, stato completato, validazione umana e righe `proposalHandling`;
- classificazione file validi, invalidi e duplicati;
- vista Referente dedicata in `EsportazioniView`;
- riepilogo di file, discipline ed esiti;
- consultazione read-only delle decisioni e delle note dipartimentali;
- nessuna validazione finale, modifica, persistenza remota, backend o telemetria.

## Confine di slice

CML-502 prepara il materiale per il Referente ma non registra ancora conferme, respinte o sospensioni. La validazione finale sarà una slice successiva.

## Validazione richiesta

- React CI;
- CML-470 Persistence Audit;
- CML-473 Comparative Audit;
- CML-491 Mobile Visual Audit;
- preview Cloudflare;
- review automatica;
- prova manuale con file valido, duplicato, formato errato e schema non compatibile.
