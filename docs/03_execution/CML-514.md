# CML-514 — Backup e ripristino

## Obiettivo

Consentire all’utente di creare un backup locale dei principali archivi CurManLight e ripristinarlo in modo esplicito, controllato e reversibile per quanto possibile.

## Implementazione

- nuovo contratto `cml-local-backup-v1`;
- backup JSON locale dei cinque archivi supportati;
- nome file con timestamp;
- selezione di un file `.json` senza scritture immediate;
- validazione di schema, app e chiavi supportate;
- anteprima con data e numero di archivi;
- conferma esplicita prima del ripristino;
- sostituzione coerente degli archivi supportati;
- rollback best-effort in caso di errore di scrittura;
- ricaricamento dell’app dopo il ripristino, così tutti gli store in memoria rileggono i dati ripristinati;
- feedback accessibili di successo o errore.

## Archivi inclusi

- Programmazioni annuali;
- UDA essenziali;
- proposte docente;
- coda e decisioni dipartimentali;
- validazioni del Referente.

## Vincoli rispettati

- nessun backend;
- nessun servizio remoto;
- nessuna autenticazione;
- nessuna telemetria;
- nessun invio di dati;
- nessun ripristino automatico;
- nessuna accettazione di chiavi locali non riconosciute;
- nessuna modifica ai dati curricolari.

## Criterio di accettazione

L’utente può scaricare un file di backup, selezionarlo successivamente, verificarne i dati essenziali e confermare il ripristino senza scritture premature. Dopo il successo, l’app viene ricaricata per impedire che store già inizializzati sovrascrivano i dati ripristinati.
