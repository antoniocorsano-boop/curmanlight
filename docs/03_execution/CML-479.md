# CML-479 — Connect Temporal Applicability Resolver to Work Context and Home Card

## Obiettivo

Collegare il resolver introdotto da CML-478 al contesto di lavoro React e mostrare nella Home quale quadro nazionale si applica alla combinazione selezionata.

La slice mantiene distinta l'applicabilità normativa dallo stato del curricolo d'istituto.

## Baseline

- `main`: `983e7a3ed21f7f81df0dcad97f7e058071330e19`
- resolver: `resolveCurriculumApplicability()`
- dataset: `curman-react/src/data/temporal-applicability.json`

## Modifiche

### Impostazioni

`curman-react/src/views/ImpostazioniView.tsx`

Aggiunge il campo **Classe o sezione**:

- Infanzia: `Tutte le sezioni`;
- Primaria: classi 1ª-5ª;
- Secondaria: classi 1ª-3ª;
- ordine `Tutti`: classe non disponibile finché non viene scelto un ordine specifico.

Il cambio di ordine azzera la classe precedente per evitare combinazioni incoerenti.

### Home

`curman-react/src/views/HomeView.tsx`

Aggiunge la card **Curricolo applicabile**, che:

- legge il profilo dal `useAppStore`;
- passa anno, ordine, classe e disciplina al resolver CML-478;
- mostra `Indicazioni nazionali 2012` oppure `Indicazioni nazionali 2025`;
- mostra il messaggio esplicativo prodotto dal resolver;
- mantiene il fallback `Applicabilità da verificare` quando il contesto è incompleto;
- offre accesso diretto alle Impostazioni.

### Separazione istituzionale

La card dichiara esplicitamente:

> Questo dato indica il quadro nazionale applicabile. Non certifica che il curricolo d'istituto sia già aggiornato, deliberato o approvato.

Questa distinzione impedisce di trasformare una regola normativa in una falsa attestazione sullo stato del curricolo dell'istituto.

## Controllo automatico

File:

```text
curman-react/tools/check-cml479-home-applicability.mjs
```

Comando:

```text
npm run test:cml479
```

Verifica:

- uso del resolver nella Home;
- presenza di entrambe le etichette normative;
- fallback prudente;
- messaggio di separazione istituzionale;
- presenza del selettore di classe;
- intervalli primaria e secondaria;
- classe obbligatoria per primaria e secondaria.

Il controllo è inserito in React CI prima della build TypeScript/Vite.

## Limiti intenzionali

CML-479 non introduce:

- selezione automatica di un dataset curricolare IN 2012 o IN 2025;
- stato di approvazione del curricolo d'istituto;
- modifica a Consultazione, Revisione o Esportazioni;
- persistenza aggiuntiva del profilo;
- backend, autenticazione o telemetria;
- deploy manuale.
