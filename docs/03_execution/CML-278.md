# CML-278 - ICON SYSTEM MINIMAL RUNTIME IMPLEMENTATION

## Tipo slice
Runtime microfix + docs

## Obiettivo
Applicare il sistema icone definito in CML-277 con implementazione runtime minima, sostituendo le emoji strutturali principali con un subset SVG locale in punti ad alta visibilita, senza modificare logiche di dominio, dati, schema `.cml`, export/import o storage.

## Scope autorizzato
- index.html
- _published_snapshot/netlify-current/index.html
- docs/03_execution/CML-278.md
- report/CML-278_icon_system_minimal_runtime_implementation.md
- docs/REPO-MOVELOG.md

## Perimetro runtime applicato
- intestazione principale (titolo e badge avviso principale)
- navigazione principale tabbar (Home, Curriculum, Competenze e progettazione, Esportazioni, Guida)
- area Home/cruscotto (azioni principali)
- pulsanti Guida rapida
- pulsante Mostra/Nascondi discipline
- card Home di ingresso operativo

## Decisioni implementative
- subset SVG locale inline (sprite simboli nel documento)
- nessun CDN, nessun font iconico remoto, nessuna dipendenza aggiuntiva
- testo sempre visibile accanto all'icona
- icone decorative con `aria-hidden="true"` e `focusable="false"`
- toggle dinamico Mostra/Nascondi discipline aggiornato via `innerHTML` con icona + testo

## Fuori scope rispettato
- nessuna modifica a dati curricolari
- nessuna modifica schema `.cml`
- nessuna modifica export/import
- nessuna modifica localStorage/sessionStorage
- nessuna modifica validatori
