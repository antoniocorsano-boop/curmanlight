# CML-374 — Curriculum Runtime Access Residual Hardcode Audit

## Mandatory Slice Header
- Macroprogramma: (autonomo — audit di consolidamento post CML-371→372→373)
- Dipende da: CML-373 (TECNOLOGIA_NORM dead code removal)
- Tipo slice: docs-only
- Perimetro: certificazione zero residui hardcoded disciplinari nei runtime files

## Obiettivo
Verificare formalmente che dopo la rimozione di `TECNOLOGIA_NORM` / `TECNOLOGIA_NORM_DATA` (CML-373) non restino altri accessi diretti, alias storici o pattern hardcoded legati a singole discipline nei due runtime files (`index.html` e `_published_snapshot/netlify-current/index.html`).

## Criteri di audit

| # | Criterio | Metodo |
|---|----------|--------|
| 1 | Zero riferimenti a `TECNOLOGIA_NORM` o `TECNOLOGIA_NORM_DATA` | `grep literal` |
| 2 | Zero pattern `*_NORM` residui (es. `MUSICA_NORM`) | `grep literal` |
| 3 | Zero accessi dot-direct a `ALL_CURRICULUM_DATA.<disciplina>` | `grep literal` |
| 4 | Zero branch `discKey === 'tecnologia'` | `grep literal` |
| 5 | Validazione curricolo JSON passa | `validate-cml-normalized-curriculum.mjs` |
| 6 | Test shape MAPPA_DATI runtime passa | `test-runtime-mappa-dati-shape.mjs` |

## Conclusioni

La catena CML-371 → CML-372 → CML-373 ha completato la migrazione dell'accesso ai dati disciplinari:

| Slice | Risultato |
|-------|-----------|
| CML-371 | Helper `MAPPA_DATI_INDEX` + `getMappaForDiscipline()` |
| CML-372 | Helper `getUnitsForDiscipline(discKey)` + rimozione 4 guardie Tecnologia |
| CML-373 | Rimozione `TECNOLOGIA_NORM` / `TECNOLOGIA_NORM_DATA` dead code |
| **CML-374** | **Audit certifica zero residui** |

L'accesso runtime alle unità di apprendimento è ora completamente generalizzato: tutte le 14 discipline transitano per `getUnitsForDiscipline(discKey)` → `ALL_CURRICULUM_DATA[discKey].unitaApprendimento`. Nessun residuo hardcoded disciplinare nei runtime files.

## Rischi residui
- Nessun residuo hardcoded nei runtime files.
- Le variabili `*_MAPPA_DATI` individuali (es. `MAPPA_DATI_italiano`) non sono state oggetto di audit — esulano dal perimetro CML-370 (accesso dati per unità di apprendimento) e riguardano la mappa dati (nuclei, competenze, obiettivi), non le unità.
- Il runtime carica ancora i dati disciplinari da `ALL_CURRICULUM_DATA` inline (16 unità per Tecnologia), mentre il file `content/curriculum/tecnologia.normalized.json` (13 unità) non è ancora caricato dinamicamente.
