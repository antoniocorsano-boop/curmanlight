# CML-413 — Clean Browser Visual Baseline Protocol

## Header

- Tipo: docs-only remoto
- Branch: main
- Baseline precedente: CML-412
- Runtime: non modificato
- Obiettivo: preparare l'acquisizione visuale reale della Pages in browser pulito

## Contesto

CML-412 ha chiarito che GitHub Pages serve il runtime corretto da `_published_snapshot/netlify-current` e che il precedente disallineamento era dovuto a vista cache o stale. Resta necessario acquisire una baseline visuale reale prima di ulteriori interventi UX.

## Decisione

La prossima valutazione UI deve basarsi su schermate reali della pagina pubblicata, non su mock e non solo su lettura del codice.

## Viste minime

- Home
- Curricolo
- Revisione
- Processo
- Finale in verifica
- Fonti
- Sezioni generali
- Progetta
- Programmazione annuale
- UDA modello
- Esportazioni
- Guida
- Menu mobile
- Modali principali visibili durante l'uso normale

## Criteri di osservazione

Per ogni vista annotare:

- titolo e percorso visibile;
- azioni principali;
- elementi chiari;
- elementi confusi;
- differenze rispetto ai mock;
- elementi da non toccare;
- possibili micro-slice successive.

## Non-obiettivi

- Non modificare runtime.
- Non introdurre nuove funzioni.
- Non usare mock come prova dello stato live.
- Non avviare test utente prima della baseline visuale.

## Esito remoto

Documento creato come protocollo docs-only. Nessun file runtime modificato.
