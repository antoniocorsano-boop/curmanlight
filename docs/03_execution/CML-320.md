## Audit — Asse Mojibake

### Emoji con variation selector danneggiato

Pattern identificato: emoji con `U+FE0F` (variation selector-16) visualizzato come `ï¸` (interpretazione Latin-1 di UTF-8).

**Occorrenze stimate nel runtime:**

| Emoji | Testo rotto | Occorrenze ~ |
|---|---|---|
| ✏️ (U+270F U+FE0F) | ✏ï¸ | ~20 |
| ⚠️ (U+26A0 U+FE0F) | ⚠ï¸ | ~12 |
| ⬇️ (U+2B07 U+FE0F) | ⬇ï¸ | ~8 |
| ⬆️ (U+2B06 U+FE0F) | ⬆ï¸ | ~4 |
| 🏛️ (U+1F3DB U+FE0F) | 🏛ï¸ | ~2 |
| ⚙️ (U+2699 U+FE0F) | ⚙ï¸ | ~2 |
| ⚖️ (U+2696 U+FE0F) | ⚖ï¸ | ~4 |
| 🏗️ (U+1F3D7 U+FE0F) | 🏗ï¸ | ~1 |
| 🗺️ (U+1F5FA U+FE0F) | 🗺ï¸ | ~2 |
| 🗓️ (U+1F5D3 U+FE0F) | 🗓ï¸ | ~3 |
| ℹ️ (U+2139 U+FE0F) | ℹï¸ | ~3 |
| 🧑‍⚖️ | 🧑‍⚖ï¸ | ~1 |
| 1️⃣ 2️⃣ 3️⃣ | 1ï¸⃣ 2ï¸⃣ 3ï¸⃣ | ~3 |

**Causa probabile**: composizione UTF-8 spezzata. Il byte sequence `EF B8 8F` (U+FE0F in UTF-8) viene interpretato come `ï¸` in Latin-1/CP1252. Il file `index.html` dichiara `<meta charset="UTF-8">` ed è salvato come UTF-8, ma i dati provenienti dai JSON curricolari potrebbero essere stati generati con encoding misto.

### Mojibake grave nei dati di Educazione Motoria

```
"nome": "Autonomia, responsabilitï¿½, espressione corporea, benessere e inclusione"
"descrizione": "Raccoglie 1 unitï¿½ di apprendimento: ..."
"Conoscenza di sï¿½, schemi motori, coordinazione di base e sicurezza negli spazi"
```

Qui `ï¿½` è il replacement character U+FFFD interpretato in Latin-1, che indica che l'encoding originale conteneva byte non validi per UTF-8. I caratteri originali sarebbero `à` (U+00E0) e `è` (U+00E8). La fonte è il JSON generato `EDUCAZIONE_MOTORIA_MAPPA_DATI` inline in `index.html`.

**Impatto**: visibile all'utente nella scheda disciplina Educazione Motoria. Non rompe funzionalità ma degrada la percezione di qualità.

### Caratteri accentati italiani corretti

I caratteri `à`, `è`, `é`, `ì`, `ò`, `ù` nei testi HTML statici (non generati da JSON) appaiono corretti. Esempi:
- `"continuità"`, `"validità"`, `"maturità"`, `"cittadinanza"` — OK
- `"non indica una nuova approvazione"` — OK
- `"puoi sempre ricominciare"` — OK

## Piano di remediation ordinato

### P0 — Correggere subito (prossima slice runtime)

1. **Riparare mojibake Education Selector su emoji** (~60 occorrenze totali in `index.html`)
   - Sostituire ogni occorrenza `ï¸` con il corretto variation selector UTF-8 `️` (U+FE0F)
   - Operazione meccanica, nessun impatto logico
   - File: `index.html` e `_published_snapshot/netlify-current/index.html`

2. **Riparare mojibake dati Educazione Motoria** (~20 occorrenze)
   - Sostituire `ï¿½` → `à` e `è` nei dati `EDUCAZIONE_MOTORIA_MAPPA_DATI`
   - Verificare anche `SCIENZE_MOTORIE_MAPPA_DATI` se presente
   - File: `index.html` e `_published_snapshot/netlify-current/index.html`

### P1 — Consolidare nella stessa slice

3. **Uniformare etichette versione curricolo**: verificare che tutte le occorrenze user-facing usino "IN 2012 (vigente)" e "IN 2025 (proposte)" coerentemente.

4. **Verificare icone disciplinari nel blocco `DISCIPLINE_META`** e rimuovere mojibake residuo (già coperto da P0.1).

5. **Verificare testi home card** post-CML-319 per coerenza.

### P2 — Rinviare a slice successiva

6. Rinominare variabili interne da `curricolo` a `curriculum` per coerenza con termine utente (solo refactoring JS, nessun impatto UI).
7. Revisione linguistica estesa dei testi generati automaticamente (formule "Raccoglie N unità di apprendimento...").
8. Uniformare `DM` vs `D.M.` in tutte le label.
9. Logo e identità visuale completi.
