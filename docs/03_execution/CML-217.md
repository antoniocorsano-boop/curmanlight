## Scopo

Implementare un pannello UDA draft minimale con preview Markdown, copia e download, generato on demand dai dati curricolari esistenti e dalle marcature evidenze locali.

## Cosa è stato implementato

### UDA Draft panel (`didattica_uda` section)

1. **Selettori**: dropdown disciplina (da DISCIPLINE), dropdown unità (da ALL_CURRICULUM_DATA)
2. **Pulsante "Genera bozza UDA"**: genera markdown on demand
3. **Pulsanti "Copia Markdown" e "Scarica Markdown"**: abilitati dopo generazione
4. **Preview Markdown**: textarea readonly con anteprima
5. **Status bar**: mostra conteggio evidenze adottate/adattate/escluse
6. **Notice box**: privacy-first, nessun dato studente, bozza non persistita

### Markdown generato

Sezioni:

- Intestazione (disciplina, competenza, ordine, nuclei, stato bozza, data)
- Riferimenti curricolari (ambito, competenza, traguardo, obiettivi, conoscenze, abilità)
- Evidenze selezionate (da `cml_evidenze_state`, escluse quelle marcate `non_applicabile`)
- Criteri di osservazione/valutazione
- Attività didattica proposta (placeholder editabile dal docente)
- Adattamenti/inclusione (placeholder)
- Nota privacy
- Fonte curricolare

### Comportamento empty state

Se nessuna evidenza è marcata: messaggio "Nessuna evidenza selezionata" + suggerimento di marcare evidenze.

### Pattern riutilizzati

- `downloadBlob()` / `downloadMarkdownString()` per download
- `copyMarkdownToClipboard()` per copia clipboard
- `getEvidenceState()` per leggere marcature da `cml_evidenze_state`
- `escapeHtml()` per sicurezza output
- `DISCIPLINE_META` per icone e nuclei
