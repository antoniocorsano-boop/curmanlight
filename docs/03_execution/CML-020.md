## Collegamento con slice precedenti

CML-018 definisce il flusso operativo generale. CML-018A predisegue il Drive condiviso Workspace. CML-019 introduce la generazione file .cml proposta docente. CML-020 aggiunge l'invio automatico al Drive.

## Obiettivo della slice

Aggiungere il pulsante "Invia al Drive condiviso" che invia automaticamente il file .cml al Drive condiviso della scuola, mantenendo sempre disponibile il download manuale come fallback.

## Funzioni aggiunte

- `uploadTeacherCmlToDrive()`: invia il file .cml all'endpoint configurato
- `getDriveUploadEndpoint()`: recupera l'endpoint da localStorage (chiave `curmanlight.driveUploadEndpoint`)

## Posizione del comando nell'interfaccia

- Toolbar "Export" nel tab Lavoro (accanto a "Scarica proposta .cml")
- Tab "Riepilogo" (accanto a "Scarica proposta .cml")

## Comportamento se invio non configurato

Se l'endpoint non è configurato, mostra il messaggio:
"ℹ️ Invio al Drive non ancora configurato. Puoi scaricare il file proposta .cml e caricarlo manualmente nel Drive condivisto della scuola."

## Comportamento se invio riuscito

Mostra il messaggio:
"✅ Proposta inviata al Drive condiviso. Il dipartimento potrà acquisirla per la validazione."

## Comportamento se invio fallito

Mostra il messaggio:
"⚠️ Non è stato possibile inviare la proposta al Drive. Puoi scaricare il file .cml e caricarlo manualmente."

## Conferma fallback manuale

Il pulsante "Scarica proposta .cml" rimane sempre visibile e funzionante, indipendentemente dallo stato dell'invio automatico.
