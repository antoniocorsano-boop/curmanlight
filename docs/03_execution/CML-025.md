# CML-025 — report per gruppo di lavoro curricolo

**Branch:** `cml-008r-fix-markdown-decision-summary`  
**Commit iniziale:** `492ff59`  
**Collegamento:** derivata da CML-024 (end-to-end workflow smoke)  
**Ottimizzazione:** aggiunta funzione di generazione report per il referente curricolo

## Obiettivo

Aggiungere una funzione per generare un **report di lavoro per il gruppo curricolo**, partendo dagli esiti dipartimentali importati dal referente.
Il report non deve essere il curricolo finale, ma un documento di lavoro utile al gruppo per:

- vedere quali dipartimenti/discipline hanno restituito esiti;
- individuare elementi già confluiti nella sintesi;
- individuare elementi riformulati;
- individuare elementi assorbiti in altra proposta;
- evidenziare elementi `da_chiarire`;
- evidenziare elementi `senza esito del confronto`;
- verificare completezza, evidenze e criticità;
- preparare la discussione collegiale.

## Funzioni aggiunte

- `buildReferentGroupWorkReportMarkdown()` – genera il contenuto Markdown del report.
- `downloadReferentGroupWorkReport()` – triggers download of the Markdown file.

## Posizione del comando nell’interfaccia

Nel pannello **Verifica referente curricolo** (id `referent-validation-panel`), dopo il riepilogo degli esiti importati, aggiunto pulsante:
`Scarica report gruppo di lavoro` (classe `export-btn btn-md`) che chiama `downloadReferentGroupWorkReport()`.

## Struttura del report Markdown

1. Titolo: `Report di lavoro per il gruppo curricolo`
2. Nota di stato: specifica che il documento è un report di lavoro, non una deliberazione finale né un curricolo approvato.
3. Sintesi generale
   - file dipartimentali importati;
   - discipline rilevate;
   - esiti totali;
   - elementi da chiarire;
   - elementi senza esito del confronto.
4. Quadro per disciplina
   - stato del lavoro dipartimentale;
   - numero proposte;
   - confluite nella sintesi;
   - riformulate dal dipartimento;
   - assorbite in altra proposta;
   - da chiarire;
   - senza esito del confronto.
5. Punti da chiarire (elenco degli elementi con `handling: "da_chiarire"`)
6. Elementi senza esito del confronto (elenco degli elementi con handling vuoto o mancante)
7. Evidenze e controlli
   - checks sui file importati (validi, non riconoscibili, duplicati);
   - discipline presenti (eventuale indicazione di discipline miste);
   - note presenti negli esiti;
   - avvisi dai file importati (duplicati, campi mancanti).
8. Traccia per la discussione del gruppo (domande guida)
9. Chiusura prudente: `La validazione finale resta a cura del gruppo di lavoro e degli organi competenti.`

## Comportamento con esiti importati

Se sono presenti esiti dipartimentali in `referentOutcomeState.outcomes`, il pulsante è abilitato e genera il report con i dati sopra descritti.
Il file scaricato ha estensione `.md` e nome consigliato `report_gruppo_curricolo_<data>.md`.

## Comportamento senza esiti importati

Se `referentOutcomeState.outcomes` è vuoto, la funzione `buildReferentGroupWorkReportMarkdown` restituisce `null` e `downloadReferentGroupWorkReport` mostra un toast:
`Importa prima almeno un esito dipartimentale .cml.`
