## Obiettivo

Rimedare il fallimento pre-esistente del runtime shape test causato da BOM nell'output della pipeline PowerShell, ripristinando 14/14 PASS.

## Root Cause

Il runtime shape test (`tools/test-runtime-mappa-dati-shape.mjs`) esegue una pipeline:

```
node tools/json-to-mappa-dati-adapter.mjs <file> | node tools/to-runtime-mappa-dati-transformer.mjs
```

Quando `execSync` usa `shell: 'powershell'`, PowerShell aggiunge un UTF-8 BOM (`U+FEFF`) all'inizio dello stream di stdout nella pipe. Il transformer (`to-runtime-mappa-dati-transformer.mjs`) legge da stdin e chiama `JSON.parse(raw)` senza strippare il BOM, causando:

```
Unexpected token '﻿', "﻿{...`
```

Il BOM **non è presente nei file JSON su disco** (verificato con `fs.readFileSync` in modalità 'utf8' e con lettura byte-by-byte). I file iniziano direttamente con `{` (byte `7B`).

## Fix

Aggiunto stripping del BOM nel transformer **prima di `JSON.parse`**:

```js
const cleaned = raw.replace(/^\uFEFF/, '').trim()
data = JSON.parse(cleaned)
```

Questa è la fix più piccola e sicura perché:

1. Non modifica alcun file di curriculum
2. Non modifica il contenuto semantico JSON
3. Rende il transformer resiliente a BOM provenienti da qualsiasi fonte (PowerShell pipe, future editor, ecc.)
4. Preserva il comportamento esistente per input senza BOM
