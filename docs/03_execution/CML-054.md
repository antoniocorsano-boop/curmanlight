## Decisione

**Opzione C — Codice operativo per dipartimento e referente**

### Condizioni

- Non chiamarlo "password" o "login"
- Usare etichetta **"codice operativo"** o **"codice di abilitazione"**
- Dichiarare esplicitamente che è un blocco locale leggero, non autenticazione
- Non impedire la consultazione del curricolo
- Memorizzare in sessionStorage, non localStorage (scompare alla chiusura scheda)
- Suggerire codice predefinito per la prova (es. `CML2025`)

### Azioni da proteggere

| Azione                                   | Ruolo        | Protezione          |
| ---------------------------------------- | ------------ | ------------------- |
| Esportazione esito dipartimentale `.cml` | Coordinatore | ✅ Codice richiesto |
| Import esiti dipartimentali (referente)  | Referente    | ✅ Codice richiesto |
| Generazione report gruppo curricolo      | Referente    | ✅ Codice richiesto |
| Reset/cancellazione dati locali          | —            | ❌ Non necessaria   |
| Import massivo                           | —            | ❌ Non presente     |

### Azioni da lasciare libere

- Lettura del curricolo (IN 2012 / IN 2025)
- Sezioni generali
- Guide, nota rilascio, scheda stato
- Consultazione esempi `.cml`
- Esportazione proposta docente `.cml`
- Import proposta docente (dipartimento)
- Validazione dipartimentale (registrazione esiti)

### Microcopy raccomandato

**Titolo:**

> Codice operativo richiesto

**Testo:**

> Questa funzione è riservata al ruolo indicato nella prova controllata. Inserisci il codice operativo fornito dal referente.

**Avvertenza:**

> Il codice evita usi accidentali dello strumento. Non è una password istituzionale e non sostituisce autorizzazioni, firme o delibere.

**Azioni pulsante:**

> "Sblocca funzione" / "Annulla"

| Rischio                                          | Mitigazione                                  |
