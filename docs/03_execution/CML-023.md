# CML-023 — REFERENT_VALIDATION_CML_IMPORT

**Branch:** `cml-008r-fix-markdown-decision-summary`  
**Commit iniziale:** `4e91441300883973e78d833385e0a26bb9ec9c8e`  
**Collegamento:** deriva da CML-019/020/021/021A/022/022A  
**Obiettivo della slice:** Aggiungere una prima modalità referente curricolo per importare manualmente gli esiti dipartimentali `.cml` prodotti dai dipartimenti, riconoscerli, verificarli e mostrarli in una vista semplice per la successiva validazione collegiale.

**Funzioni aggiunte:**
- `importDepartmentOutcomeCmlFiles()` (async)
- `validateDepartmentOutcomeCmlModel()`
- `parseDepartmentOutcomeCmlText()`
- `renderReferentValidation()`
- `buildReferentValidationSummary()`

**Posizione del comando nell’interfaccia:**
Nel tab `Revisione`, sotto la sezione `Validazione dipartimentale`, nuovo pannello:
`Verifica referente curricolo`
con pulsante `Importa esiti dipartimentali .cml`.

**Schema minimo validato:**
{
  "fileType": "department_outcome",
  "humanValidationRequired": true,
  "disciplineWorkStatus": "completed",
  "proposalHandling": [
    {
      "proposalId": "...",
      "handling": "confluita_nella_sintesi",
      "note": ""
    }
  ],
  "checks": {
    "hasHandling": true
  }
}

**Comportamento file valido:**
- Viene accettato se supera la validazione JSON e lo schema minimo.
- Viene aggiunto allo stato `referentOutcomeState` e visualizzato nel riepilogo e nelle card.

**Comportamento file non valido:**
- Viene rifiutato e segnalato nel feedback con il codice di errore appropriato (JSON non valido, tipo file errato, dati mancanti, etc.).

**Riepilogo referente:**
Dopo l’import, viene mostrato un riepilogo con:
- File importati
- File validi
- File non riconoscibili
- Esiti totali
- Discipline rilevate
- Proposte da chiarire (handling: "da_chiarire")
- Proposte senza esito (handling mancante o vuoto)

**Controlli disponibili:**
Nella vista delle card, per ogni esito vengono mostrati:
- Disciplina (escaped)
- Stato lavoro dipartimentale (escaped)
- Numero di proposte gestite per ogni stato (confluita_nella_sintesi, riformulata_dal_dipartimento, assorbita_in_altra_proposta, da_chiarire, senza esito)
- Nota (se presente, escaped)
- Controlli (hasHandling, etc.)

**Conferma import locale:**
L’import avviene tramite FileReader, nessuna chiamata di rete, nessuna persistenza permanente (salvo lo stato temporaneo già esistente).

**Confini rispettati:**
- Nessuna chiamata di rete durante l’import.
- Nessuna introduzione di Google API, Apps Script, endpoint reali, autenticazione.
- Nessuna modifica allo schema `department_outcome` (se non per la validazione esistente).
- Nessuna semantica rigida di accoglimento/respingimento.
- Nessuna modifica a PDF, `sw.js`, `_headers`, asset non necessari.
- Nessun deploy.
