## Analisi

### Errore 1 — `Unexpected identifier 'open'`

- L'errore viene rilevato più volte nella console del deploy.
- **Non riproducibile** con `node` sul sorgente locale: lo script inline di `_published_snapshot/netlify-current/index.html` passa il controllo sintattico Node.js.
- Il file deployato è più piccolo di quello locale (`823358` vs `833140` byte), quindi GitHub Pages potrebbe servire una versione precedente o troncata.
- La stringa `open` è usata ampiamente come **classe CSS** (`.open`) e come **nome di variabile locale** (`const open = ...`); nessun uso come identificatore standalone nello script valido.
- Nessun eval/new Function/script inline sospetto oltre allo script principale.
- Impatto: **non bloccante confermato** su Home; valutazione in corso per sezioni Curriculum/Guida/Processo/Esportazioni/UDA smart.

### Errore 2 — `docx.umd.min.js 404`

- Riferimento CDN esterno: `https://cdnjs.cloudflare.com/ajax/libs/docx/8.5.0/docx.umd.min.js`
- Lo script è marcato come opzionale nel markup; nel codice è presente un guard `if(typeof docx==="undefined")`.
- L'export Word/docx dipende da questa libreria; senza di essa l'export Word è disabilitato ma altre esportazioni rimangono disponibili.
- Priorità: **P1** se l'export Word è nel perimetro UX, altrimenti **P2**.

### Errore 3 — `favicon.ico 404`

- Manca il file `/favicon.ico` alla root del sito.
- Non blocca navigazione, PWA o tab.
- Priorità: **P3**
