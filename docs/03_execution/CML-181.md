## Sintesi checklist

| Area | Domande chiave |
|---|---|
| Identita disciplinare | Nome, collocazione, lessico ufficiale |
| Nuclei | Confermare/modificare/unire/eliminare i 5 nuclei proposti |
| Progressione I/II/III | Distinguere specifico, ricorsivo, complessita crescente |
| Obiettivi osservabili | Azioni motorie, collaborazione, regole, sicurezza |
| Traguardi | Coerenza con nuclei e obiettivi, verificabilita |
| Conoscenze/abilita/atteggiamenti | Separare saperi, abilita, comportamenti |
| Valutazione | Criteri osservabili, livelli, evidenze, inclusione |
| Inclusione e sicurezza | Adattamenti, spazi, prevenzione, linguaggio |
| Compatibilita CML | Campi popolabili, prudenti, da lasciare vuoti |
| Decisione finale | Sufficiente/parziale/insufficiente per data-prep |

## Esiti possibili

1. **Validazione sufficiente**: si puo autorizzare una data preparation controllata (limitata e prudente).
2. **Validazione parziale**: serve integrazione documentale prima della data-prep.
3. **Validazione insufficiente**: data preparation bloccata; serve revisione sostanziale o fonte aggiuntiva.
