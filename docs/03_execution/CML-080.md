# CML-080 — CURRICULUM AND DIDATTICA COMPLETION SELECTION AUDIT

## Context

CurManLight has reached a stable UI: professional layout, dark mode, normalized icons, improved referent report, operational microguide, and the `.cml` workflow (docente → dipartimento → referente) are all live. The next priority is **content value**: making the two central areas — Curriculum and Didattica — fully usable for real school workflow.

## Current State Assessment

### Curriculum area (4 sub-tabs under "Curriculum" tab)

| Sub-area | Status | Notes |
|---|---|---|
| **Consulta** (`tab-curricolo`) | ✅ Working | Version toggle IN 2012 ↔ 2025, all disciplines rendered with traguardi + obiettivi, index navigation, reference section |
| **Tecnologia normalizzata** | ✅ Working | Full normalized data package (CML-062) with units, traguardi, obiettivi, conoscenze, abilità, evidenze, criteri |
| **Revisione** (`tab-lavoro`) | ✅ Working | Full revision workflow: filters, decision buttons, department import, referent validation, export |
| **Definitivo** (`tab-riepilogo`) | ✅ Working | Approved+unchanged items view, all export formats |
| **Fonti** (`tab-normativa` + `tab-generali`) | ✅ Working | Complete institutional sources, PTOF, RAV, IN 2012/2025, EU frameworks, general sections |

**Gap**: The transition from consultazione → revisione → decisione is technically functional but not explicitly guided within the Consulta view itself. The user lands on "Consulta" and must know to navigate to "Revisione" sub-tab to act. The distinction between "documento vigente", "proposta di aggiornamento", "decisione dipartimentale", and "sintesi referente" is implicit in the tab structure but not explained in-context.

### Didattica area (under "Didattica" tab)

| Sub-area | Status | Notes |
|---|---|---|
| **Valutazione / Evidenze** | ⚠️ Prototype | Read-only, links Tecnologia units to evidence + criteria, filterable by order |
| **UDA modello** | ⚠️ Prototype | Read-only, shows one UDA template from Tecnologia |
| **Prepara attività** | ❌ Not implemented | Marked "In preparazione" in home card |
| **Materiali per la classe** | ❌ Not implemented | Marked "In preparazione" in home card |

**Gap**: The Didattica area is explicitly marked as prototype/"in costruzione". It has value for demonstration but lacks the operational depth a teacher needs: no guidance on how to use the evidence in grading, no connection to classroom activities, no concrete examples beyond the single UDA template.

## Options

### A — Complete Curriculum first (recommended)
Aligns with the primary workflow (revisione curricolo) and the `.cml` flow already in place. Focus on:
- Adding explicit guidance within Consulta view showing the path from consult → revise → decide → export
- Clarifying the four states (vigente, proposta, decisione, sintesi) with visual markers
- Making the "next action" clear for each discipline
- Adding a status summary in the Curriculum tab itself

**Risk**: Low
**Effort**: Medium

### B — Complete Didattica first
Would serve teachers directly, but:
- The curriculum revision workflow is the primary institutional mandate
- Expanding Didattica before Curriculum is complete risks scope creep
- The `.cml` flow depends on Curriculum being solid first

**Risk**: Medium
**Effort**: High

### C — Complete both in one slice
Too many changes at once; hard to verify and may destabilize the runtime.

**Risk**: High
**Effort**: Very high

### D — Audit first, then separate runtimes (recommended approach)
Same as what we're doing: CML-080 as selection audit, then CML-081/082 for Curriculum, CML-083/084 for Didattica.

## Recommendation

**Option A** — Complete Curriculum first, in sequence:
1. CML-080 → selection audit (this document)
2. CML-081 → Curriculum area completion runtime increment
3. CML-082 → Curriculum area live smoke
4. CML-083 → Didattica area completion runtime increment
5. CML-084 → Didattica area live smoke

## Principle

- **Curriculum** answers: *What must be updated in the school's curriculum?*
- **Didattica** answers: *How can the teacher use these guidelines in their teaching?*

They remain separate. They must dialogue, not merge.
