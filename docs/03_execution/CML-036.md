## 4. Nota operativa Netlify

Il comando `npx netlify deploy --prod --dir _published_snapshot/netlify-current` fallisce con **Forbidden** sul piano free tier Netlify, probabilmente per limitazioni del piano o della configurazione del sito (`prevent_non_git_prod_deploys`).

**Workaround**: eseguire `npx netlify deploy --dir _published_snapshot/netlify-current` (draft deploy), quindi promuovere a produzione via API REST `restoreSiteDeploy` con autenticazione token.

Esempio:

```
$ npx netlify deploy --dir _published_snapshot/netlify-current
  → Deploy ID: <deploy_id>
$ npx netlify api restoreSiteDeploy --data '{ "site_id": "<site_id>", "deploy_id": "<deploy_id>" }'
```

**Raccomandazione**: nei prossimi CML, verificare il metodo di deploy supportato prima di tentare `--prod`. L'errore CLI Forbidden non è un errore runtime dell'app.
