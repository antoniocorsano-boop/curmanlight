# CML-504 — Referent Validation Export

## Obiettivo

Completare il flusso Referente React generando localmente un file `.cml` di tipo `referent_validation` a partire dalle validazioni registrate in CML-503.

## Implementazione

- builder dedicato `referentValidationExport.ts`;
- inclusione delle sole validazioni associate agli esiti dipartimentali selezionati;
- riepilogo per confermate, respinte e sospese;
- riferimenti ai file `department_outcome` sorgente;
- esclusione dichiarata degli esiti ancora senza validazione;
- pannello export dedicato nell’area Referente;
- controllo del codice operativo prima della lettura e del download;
- download locale `.cml` con `fileType: referent_validation`;
- `humanValidationRequired: true`;
- nessun backend, upload o telemetria.

## Nota operativa

Il pannello di export richiede di selezionare nuovamente gli stessi file `department_outcome.cml`. Le validazioni locali vengono riconosciute tramite fingerprint del file e `proposalId`, evitando l’esportazione di dati orfani appartenenti a precedenti sessioni.

## Validazione richiesta

- React CI;
- CML-470 Persistence Audit;
- CML-473 Comparative Audit;
- CML-491 Mobile Visual Audit;
- preview Cloudflare;
- review automatica;
- prova manuale con validazioni complete, parziali e nessuna validazione.

## Verdetto atteso
