## Obiettivo

Definire un modello stabile per ricevere, leggere e trasformare le osservazioni dei docenti raccolte tramite il test guidato CML-400 e il pilot kit CML-428.

La slice evita interventi casuali sulla UI: prima struttura le evidenze, poi abilita decisioni di roadmap motivate.

## Ambito

Incluso:

- modello di intake dei file `.txt` esportati dai docenti;
- classificazione delle evidenze qualitative;
- criteri di severità e priorità;
- regole per trasformare osservazioni in nuove slice CML;
- schema di registro interno per il referente;
- criteri di esclusione per richieste fuori perimetro.

Escluso:

- modifiche runtime;
- modifiche dati curricolari;
- backend, account, moduli remoti o raccolta centralizzata;
- modifiche al formato `.cml` o all'import/export.
