# CML-260 — MULTI-AGENT ROLE-OPERATIVE LANGUAGE AND COGNITIVE LOAD BACKLOG AUDIT

## Contesto post CML-259P

CML-259P ha pushato l'audit di stabilità finale (commit `fb306dd`). Il ciclo UX CML-246 → CML-259P è chiuso con decisione `UX_CYCLE_STABLE_AND_CLOSED`.

Tuttavia, durante il ciclo sono emersi pattern di linguaggio tecnico residuo (JSON visibile, anglicismi, underscore nei valori UI), disallineamenti di ruolo (Dirigente assente, Gruppo di lavoro passivo) e carico cognitivo alto in alcune superfici (Home, Processo, Esportazioni, Competenze).

CML-260 è un audit multi-agente che analizza in parallelo 5 dimensioni del runtime per produrre un backlog consolidato di miglioramenti linguaggio-ruolo-carico cognitivo.

## Obiettivo dell'audit

1. **Surface Language (Agente 1)** — trovare termini tecnici visibili in UI ordinaria
2. **Role Journey (Agente 2)** — valutare chiarezza del percorso per ogni ruolo scolastico
3. **Cognitive Load (Agente 3)** — identificare sovraccarico informativo per schermata
4. **Export/File Naming (Agente 4)** — valutare chiarezza delle etichette di esportazione
5. **School Language (Agente 5)** — analizzare coerenza con lessico scolastico reale

### Agente 1 — Surface Language Scanner

**Tipo:** Ricerca superfici runtime (grep + read su `index.html`)

**Findings: 20 totali**

| Priority | Count | Esempi |

### Agente 2 — Role Journey Analyzer

**Tipo:** Analisi percorso ruolo per ruolo nella Guida e nel Processo

**Findings:**

| Ruolo | Chiarezza | Problema |
|-------|-----------|----------|
| Docente | Alta | Percorso chiaro: consulta → programma → UDA |
| Dipartimento | Media | "Confronta" vago, non specifica come/dove |
| Referente | Alta (testualmente) | Role access gate può bloccare |
| **Dirigente** | **CRITICA** | **Completamente assente** dall'interfaccia |
| Gruppo di lavoro | Bassa | Ruolo solo passivo, nessuna azione |

**Key gaps:**
- Nessuna selezione ruolo nel profilo
- Documento finale manca di owner esplicito
- Dirigente: nessun riferimento in nessuna schermata

### Agente 3 — Cognitive Load Analyst

### Agente 4 — Export/File Naming Auditor

**Tipo:** Analisi etichette e flusso esportazioni

**Findings: 10 critici**

| Issue | Descrizione |
|-------|-------------|
| 1 | "Resoconto sintesi" non è un export (è una view) |
| 2 | "Copia per Word" suggerisce editing, ma produce .docx read-only |
| 3 | "Testo" ambiguo — non dice cosa contiene |
| 4 | Etichetta `.cml` solo negativa, mai spiegata positivamente |
| 5 | Guida ruoli piatta, non sequenziale per fase |
| 6 | Nessuna distinzione tra export singolo e multiplo |
| 7 | "Report" in contesto scolastico non chiaro |
| 8 | "Backup" e "Archivio" sovrapposti |
| 9 | Formato file mai esplicitato prima del download |
| 10 | Nessuna etichetta "Cosa ottieni" per ogni export |

### Agente 5 — School Language Consistency Auditor

**Tipo:** Analisi terminologia vs lessico scolastico reale

**Findings:**

| Priority | Count | Esempi |
|----------|-------|--------|
| P0 | 2 | "UDA smart" (12+ occorrenze), "persistita" (non italiano scolastico) |
| P1 | 5 | "Reset", "report" (3x), "import"/"export" come nomi inglesi |
| P2 | 15 | Voce passiva, tono burocratico, incongruenze microcopy |

**Style dictionary proposto:** 10 regole di sostituzione linguistica (es. "UDA smart" → "UDA personalizzata", "persistita" → "salvata", "reset" → "ripristina", "report" → "documento"/"scheda")

## Roadmap proposta (CML-261+)

### CML-261 — Role Language Surface Cleanup (microcopy-only)
**Tipo:** runtime microfix
**Priorità:** Alta
**Stima:** ~20 righe modificate in index.html + snapshot
**Fix:**
- Sostituire "UDA smart" con "UDA personalizzata" (12+ occorrenze)
- Sostituire "persistita" con "salvata"
- Sostituire "Reset" con "Ripristina"

- Sostituire "import"/"export" come nomi con "importa"/"esporta"

### CML-262 — Export Naming Operational Microcopy (runtime microfix)
