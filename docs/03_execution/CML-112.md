# CML-112 — DISCIPLINARY_HASH_SELECTION_AND_TITLE_GENERALIZATION_FIX

## Contesto
- Branch: `main`
- Commit iniziale: `40b3084` (CML-111)
- Bug P0: hash `#cur-{disciplina}` non sincronizzato con `selDisc`
- Causa: `selectDisc()` non aggiornava `location.hash`; nessun listener `hashchange`; INIT non leggeva hash

## Modifiche a `index.html` (14 insertions, 1 deletion)

### 1. `resolveDiscFromHash()` — nuova funzione (linea 2299)
- Legge `location.hash`
- Riconosce pattern `#cur-{slug}` via regex `/^#cur-(.+)$/`
- Cerca disciplina corrispondente in `DISCIPLINE` tramite slug (spazi/parentesi rimossi)
- Restituisce nome disciplina o `null`

### 2. `selectDisc(d)` — aggiornata (linea 2305)
- Imposta `location.hash = '#cur-' + slug` dopo aver aggiornato `selDisc`
- `renderSidebar()` e `render()` già chiamati — invariati

### 3. Listener `hashchange` — nuovo (linea 2309)
- Legge disciplina dall'hash tramite `resolveDiscFromHash()`
- Se diversa da `selDisc`, chiama `selectDisc()`
- Guard `d!==selDisc` previene loop

### 4. INIT — aggiornato (linea 4754)
- Prima di `renderProfileSummary()`, legge hash e imposta `selDisc` se valido
- Fallback: se hash non disciplinare, `selDisc` resta "Tecnologia" (default storico)

### 5. `renderTecnologiaNorm()` — titolo dinamico (linea 4729)
- `'Tecnologia — pacchetto curricolare normalizzato'` → `' + selDisc + ' — pacchetto curricolare normalizzato'`
- Il titolo ora riflette la disciplina attiva
