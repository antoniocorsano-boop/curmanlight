# CML-512 — Esportazione progettazione

## Obiettivo

Consentire al docente di scaricare localmente una Programmazione annuale o un’UDA essenziale già salvata, senza introdurre backend, servizi remoti o nuovi contratti `.cml` di processo.

## Stato iniziale

- CML-508 Programmazione annuale: integrata.
- CML-509 UDA essenziale: integrata.
- CML-510 riutilizzo curricolare: integrata.
- CML-511 salvataggio e duplicazione: integrata.
- Gli store locali restano separati e versionati.

## Contratto di esportazione

L’esportazione è disponibile solo quando il documento corrente risulta salvato e il contesto contiene disciplina, ordine, anno scolastico e classe.

Formati introdotti:

1. **Markdown `.md`**
   - leggibile e modificabile;
   - contiene metadati, sezioni compilate e identificativi curricolari;
   - include il richiamo alla validazione professionale.

2. **JSON `.json`**
   - envelope `cml-planning-export-v1`;
   - distingue `annual_plan` e `uda`;
   - conserva il documento salvato, data di esportazione e obbligo di validazione umana.

I file di progettazione non utilizzano l’estensione `.cml`, riservata ai contratti del flusso Docente → Dipartimento → Referente.

## Implementazione

Il pannello condiviso `DuplicateDocumentPanel` diventa una superficie unica per:

- duplicazione verso una classe differente;
- download Markdown;
- download JSON;
- feedback accessibile di successo o errore.

Il documento esportato viene recuperato dallo store locale tramite la stessa chiave composta già usata dalle viste: disciplina, ordine, anno scolastico e classe.

## Sicurezza e privacy

- generazione interamente nel browser;
- nessun invio dati;
- nessuna telemetria;
- nessuna autenticazione;
- nessun caricamento remoto;
- revoca immediata dell’Object URL dopo il download.

## Compatibilità

- nessuna migrazione degli store;
- nessuna modifica ai record CML-508, CML-509, CML-510 o CML-511;
- contenuti e `sourceUnitIds` restano invariati;
- duplicazione e salvataggio continuano a usare le chiavi esistenti.

## Verifiche richieste

- TypeScript e build;
- export Markdown da Programmazione annuale;
- export JSON da Programmazione annuale;
- export Markdown da UDA;
- export JSON da UDA;
- blocco prima del salvataggio;
- metadati e riferimenti curricolari presenti;
- nomi file normalizzati;
- nessuna modifica al salvataggio dopo il download;
- `git diff --check`;
- workflow GitHub e verifica browser.

## Limiti residui

- nessuna esportazione PDF o DOCX in questa slice;
- nessuna reimportazione dei file di progettazione;
- nessun archivio multi-documento per più copie nella stessa classe;
- nessuna firma o approvazione automatica.

## Verdetto atteso
