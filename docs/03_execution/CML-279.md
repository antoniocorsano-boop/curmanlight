# CML-279 — Work Context Lightweight Runtime Entry

## Obiettivo

Introdurre nella Home una sezione leggera **Contesto di lavoro** che consenta all'utente di:
- consultare liberamente il curricolo senza contesto;
- impostare un contesto non sensibile (ruolo, anno, classe, disciplina opzionale, base curricolare);
- visualizzare un riepilogo del contesto attivo;
- modificare o reimpostare il contesto.

## Contesto

- CML-269 ha definito il Work Context Layer.
- CML-276 ha ridisegnato la Home come cruscotto leggero.
- CML-278 ha introdotto il sistema icone SVG locale.

## Intervento runtime

### 1. CSS (`index.html`)

Aggiunte classi:
- `.home-context-card` — card principale con bordo sinistro #00695c
- `.home-context-title` / `.home-context-body` / `.home-context-empty`
- `.home-context-actions` / `.home-context-primary` / `.home-context-danger`
- `.home-context-summary` — riepilogo contesto attivo
- `.work-context-overlay` / `.work-context-dialog` — modal
- `.work-context-field` / `.work-context-actions` — form
- `.work-context-primary` / `.work-context-secondary` — pulsanti

### 2. SVG icons

Aggiunti due nuovi simboli allo sprite locale:
- `icon-briefcase` — per la card contesto
- `icon-arrow-right` — per azione "Consulta liberamente"

### 3. HTML

Aggiunta sezione `<section class="home-context-card">` nella Home dashboard, dopo la griglia `home-cards` e prima del pannello discipline. Il contenuto è popolato via JS.

### 4. JavaScript

Nuove costanti e funzioni:

| Costante / Funzione | Descrizione |
|---|---|
| `WORK_CONTEXT_KEY` | `"cml_work_context_v1"` — chiave localStorage |
| `WORK_CONTEXT_ROLES` | Ruoli disponibili (5 opzioni) |
| `WORK_CONTEXT_BASES` | Basi curricolari (5 opzioni) |
| `CURRENT_SCHOOL_YEAR` | `"2026/2027"` |
| `WORK_CONTEXT_CLASSES` | Classi disponibili (9 opzioni) |
| `loadWorkContext()` | Legge contesto da localStorage |
| `saveWorkContext(data)` | Salva contesto in localStorage |
| `resetWorkContext()` | Rimuove contesto con conferma |
| `openWorkContextPanel()` | Mostra modal di impostazione |
| `closeWorkContextPanel()` | Chiude modal |
| `applyWorkContext()` | Salva dati dal form e aggiorna UI |
| `renderWorkContextSummary()` | Aggiorna la card contesto nella Home |

Integrazione:
- Chiamata `renderWorkContextSummary()` in `setTab` per il tab home
- Chiamata `renderWorkContextSummary()` nell'inizializzazione (dopo `loadLocalState()`)

### Dati salvati (localStorage)

```json
{
  "ruolo": "docente",
  "anno": "2026/2027",
  "classe": "seconda",
  "disciplina": "Tecnologia",
  "base": "vigente",
  "savedAt": "2026-07-03T..."
}
```

Nessun dato personale. Solo dati generici non sensibili.
