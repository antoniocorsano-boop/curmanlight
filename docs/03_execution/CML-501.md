# CML-501 — Department Outcome Export

## Obiettivo

Completare il flusso dipartimentale React consentendo il download locale di un file `.cml` di tipo `department_outcome` costruito dalle decisioni registrate nella coda CML-500.

## Ambito

- export esclusivamente locale;
- nessun backend, upload, autenticazione o telemetria;
- inclusione delle sole decisioni complete;
- esclusione esplicita delle proposte ancora da decidere;
- mantenimento di `humanValidationRequired: true`;
- nessuna validazione del Referente, rinviata alla fase successiva.

## Implementazione

### Builder dedicato

File: `curman-react/src/lib/departmentOutcomeExport.ts`

Responsabilità:

- calcolo dello stato di prontezza;
- filtro delle decisioni complete;
- costruzione del contratto `CmlDepartmentOutcome`;
- conteggi per esito;
- normalizzazione del testo finale;
- conservazione dei riferimenti alla proposta e al file docente sorgente;
- generazione di un nome file leggibile;
- download tramite `downloadCml`.

### Regole del testo finale

- `accettata`: testo proposto dal docente;
- `modificata`: testo concordato dal Dipartimento;
- `respinta`: testo vigente conservato come riferimento;
- `rinviata`: nessun testo finale definitivo.

### Compatibilità Referente

Il file esportato mantiene il contratto v1 utilizzato dal percorso Referente legacy:

- `discipline` è sempre valorizzato;
- `disciplineWorkStatus` è impostato a `completed`;
- `proposalId` usa l'identificativo originario della proposta docente, con fallback a `unitaId`;
- gli esiti React sono tradotti nei valori `handling` compatibili con il contratto v1;
- la decisione semantica React resta disponibile in `sourceProposals`.

Mappatura:

- `accettata` → `confluita_nella_sintesi`;
- `modificata` → `riformulata_dal_dipartimento`;
- `respinta` → `assorbita_in_altra_proposta`;
- `rinviata` → `da_chiarire`.

### Interfaccia

La coda dipartimentale mostra:

- decisioni esportabili;
- proposte ancora da decidere;
- conteggi per esito;
- eventuali decisioni incomplete;
- pulsante `Esporta esito dipartimentale` abilitato solo quando esiste almeno una decisione completa e non ci sono decisioni modificate invalide.

## Contratto prodotto

Il file generato contiene almeno:

- `schemaVersion: "1.0"`;
- `fileType: "department_outcome"`;
- `appName: "CurManLight"`;
- `role: "department"`;
- `discipline` e `disciplineWorkStatus` compatibili con l'import Referente;
- discipline, ordine e anno scolastico;
- `proposalHandling`;
- `summary`;
- riferimenti sorgente in `sourceProposals`;
- controlli preventivi in `checks`;
- `humanValidationRequired: true`.
