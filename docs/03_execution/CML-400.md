- Perimetro: tradurre i mock in una struttura reale compatibile con CurManLight

## Obiettivo

Definire e salvare nel repository una struttura implementabile della Home come **hub del processo curricolare completo**, non limitato a consultazione e progettazione didattica.

La Home deve rendere leggibile l'intero ciclo:

```text
fonti ufficiali → consultazione → revisione/proposta → confronto dipartimentale → validazione referente → approvazione collegiale → esportazioni
```

Il percorso operativo resta:

```text
consulta → scegli → progetta → esporta → verifica
```

ma deve essere collegato esplicitamente al processo istituzionale di origine, revisione, validazione e approvazione del curricolo.

## Lettura del sistema attuale

La pubblicazione live conferma che CurManLight espone già più aree oltre alla consultazione:

- navigazione principale: Home, Curricolo, Progetta, Esportazioni, Guida;
- sottosezioni del curricolo: Consulta, Revisione, Processo, Definitivo, Fonti, Sezioni generali;
- aree di progettazione: Valutazione/Evidenze, Programmazione annuale, UDA modello;
- esportazioni per ruolo e fase: Documento finale, Confronto modifiche, Bozza disciplina, File `.cml`, Documento di sintesi, Copia di sicurezza;
- guida operativa in 5 passi;
- processo umano Docente → Dipartimento → Referente → Collegio;
- dati locali, nessun server, nessun invio automatico a Drive.

## Decisione di implementazione

Non si procede con un redesign totale. La struttura proposta è una **evoluzione della Home**:

1. mantenere l'architettura single-file esistente;
2. non modificare dati curricolari;
3. non modificare schema `.cml`;
4. non introdurre dipendenze;
5. aggiungere un blocco Home autoesplicativo sul processo completo;
6. rendere visibile il legame tra ruoli, file `.cml`, validazione umana ed esportazioni;
7. preservare il percorso già consolidato: consulta → scegli → progetta → esporta → verifica.

## Struttura runtime proposta

### 1. Hero operativo della Home

Titolo consigliato:

```text
Hub del processo curricolare
```

Testo:

```text
CurManLight accompagna l'intero ciclo del curricolo: fonti ufficiali, consultazione, revisione, progettazione, esportazioni e validazione umana.
```

CTA primarie:

- `Consulta il curricolo` → `setTab('curricolo')`
- `Apri processo` → `setTab('curricolo'); setCurricoloView('processo')` se disponibile, altrimenti ancoraggio/CTA verso Curricolo
- `Vai a Progetta` → `setTab('didattica')`

### 2. Percorso operativo

Blocchi orizzontali:

1. **Consulta** — fonti, curricolo vigente, sezioni generali;
2. **Scegli** — disciplina, ordine, classe/fascia, unità;
3. **Progetta** — evidenze, programmazione annuale, UDA;
4. **Esporta** — documento finale, confronto, bozza, `.cml`, copia;
5. **Verifica** — processo, referente, Collegio, validazione umana.

### 3. Pipeline istituzionale

Mini pipeline in Home:

```text
Docente → Dipartimento → Referente → Collegio
```

Con file/uscite:

- Docente: proposta `.cml`;
- Dipartimento: esito condiviso;
- Referente: controllo coerenza e validazione;
- Collegio: documento approvato.

### 4. Aree del sistema

Card principali:

- **Curricolo** — Consulta, Revisione, Processo, Definitivo, Fonti, Sezioni generali;
- **Progetta** — Valutazione/Evidenze, Programmazione annuale, UDA modello;
- **Esportazioni** — Documento finale, Confronto modifiche, Bozza disciplina, File `.cml`, Copia;
- **Guida** — percorso operativo e istruzioni per ruolo.

### 5. Principi sempre visibili

Tre badge leggeri:

- **Dati locali** — tutto resta sul dispositivo;
- **Nessun invio automatico** — Drive solo caricamento manuale;
- **Validazione umana** — nessuna approvazione automatica.
