# CML-497 — Teacher Proposal `.cml` Export

## Obiettivo

Consentire al docente di esportare una proposta già salvata come file locale `.cml`, senza backend e senza inviare dati all’esterno.

## Modifiche

- riuso del contratto `.cml` v1.0 già adottato da CurManLight;
- `fileType: teacher_proposal` e `proposals[]` compatibili con il parser React;
- campi `discipline`, `counts` e `humanValidationRequired` compatibili con l’importatore dipartimentale legacy;
- proposta esportata con identificativo, ordine, campo modificato, testo vigente, testo proposto, motivazione, fonte e note;
- contesto utente incluso quando disponibile;
- serializzazione JSON leggibile dentro file `.cml`;
- nome file normalizzato con disciplina e identificativo dell’unità;
- pulsante `Esporta .cml` disponibile solo per una bozza già salvata;
- nessuna modifica al curricolo vigente;
- nessuna modifica alla persistenza locale esistente;
- nessun backend, upload o telemetria.

## Contratto del file

```json
{
  "schemaVersion": "1.0",
  "fileType": "teacher_proposal",
  "appName": "CurManLight",
  "createdAt": "ISO-8601",
  "role": "teacher",
  "discipline": "Disciplina",
  "ordine": "Primaria o Secondaria",
  "annoScolastico": "",
  "profilo": {},
  "counts": {
    "total": 1,
    "ok": 0,
    "modifica": 1,
    "nuovo": 0
  },
  "proposals": [],
  "humanValidationRequired": true
}
```
