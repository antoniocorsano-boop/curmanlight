# CML-497 — Teacher Proposal `.cml` Export

## Obiettivo

Consentire al docente di esportare una proposta già salvata come file locale `.cml`, senza backend e senza inviare dati all’esterno.

## Modifiche

- introdotto il contratto file `cml-teacher-proposal-v1`;
- introdotto il tipo logico `teacher_proposal`;
- serializzazione JSON leggibile dentro file `.cml`;
- MIME type dedicato `application/vnd.curmanlight.proposal+json`;
- nome file normalizzato con disciplina e identificativo dell’unità;
- pulsante `Esporta .cml` disponibile solo per una bozza già salvata;
- nessuna modifica al curricolo vigente;
- nessuna modifica alla persistenza locale esistente;
- nessun backend, upload o telemetria.

## Contratto minimo del file

```json
{
  "schema": "cml-teacher-proposal-v1",
  "kind": "teacher_proposal",
  "exportedAt": "ISO-8601",
  "product": "CurManLight React",
  "proposal": {}
}
```

## Criteri di verifica

- React CI;
- CML-470 Persistence Audit;
- CML-473 Comparative Audit;
- CML-491 Mobile Visual Audit;
- preview Cloudflare;
- verifica che l’azione compaia solo dopo il salvataggio della bozza.

## Esclusioni

L’importazione e la riapertura del file saranno trattate nella fase dipartimentale successiva, a partire da CML-498.

`CML_497_TEACHER_PROPOSAL_CML_EXPORT_READY_FOR_REMOTE_VALIDATION`
