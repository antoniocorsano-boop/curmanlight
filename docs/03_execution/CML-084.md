# CML-084 — DISCIPLINARY_CURRICULUM_COMPLETENESS_AND_APPROVAL_READINESS_AUDIT

## Contesto

- CML-081/081A: area Curriculum completata.
- CML-082/082A: area Didattica completata.
- CML-083B/083C: design system riportato a light mode only.
- Prima di introdurre stati di approvazione, è emerso che solo Tecnologia possiede una struttura normalizzata completa.

## Problema

La UI di Curriculum mostra 15 discipline come equivalenti nella lista di consultazione, ma i dati sottostanti sono profondamente disomogenei. Tecnologia è l'unica disciplina con un pacchetto normalizzato pilota. Le altre discipline hanno solo una struttura base (traguardi + obiettivi) senza:

- ambiti organizzati;
- conoscenze/abilità esplicitate;
- evidenze osservabili;
- criteri di valutazione;
- unità di apprendimento modello.

Introdurre stati di approvazione senza aver prima verificato la completezza strutturale esporrebbe a:

- far apparire discipline incomplete come "pronte per approvazione";
- generare bozze curricolari prive di contenuti essenziali;
- delegare al runtime distinzioni che dovrebbero essere strutturali.

## Criterio di completezza

Una disciplina è considerata **strutturalmente completa** se possiede:

1. Traguardi per tutti gli ordini pertinenti (Infanzia, Primaria, Secondaria) — LIVELLO BASE
2. Obiettivi per tutti gli ordini — LIVELLO BASE
3. Ambiti organizzati per unità — LIVELLO NORMALIZZATO
4. Conoscenze esplicitate — LIVELLO NORMALIZZATO
5. Abilità esplicitate — LIVELLO NORMALIZZATO
6. Evidenze osservabili — LIVELLO NORMALIZZATO
7. Criteri di valutazione — LIVELLO NORMALIZZATO
8. Unità apprendimento modello — LIVELLO UDA
9. File normalizzato dedicato su filesystem — LIVELLO FILE

## Impatto sul futuro flusso approvazione

1. Il flusso di approvazione (CML-085/086) DEVE distinguere tra discipline con struttura completa e discipline solo consultabili.
2. Tecnologia potrà essere l'unica disciplina a mostrare stati di approvazione significativi (es. "bozza generabile" → "in revisione").
3. Le altre 14 discipline dovranno mostrare un indicatore "struttura da completare" o "solo consultazione".
4. Non è possibile generare una bozza curricolare disciplinare completa per discipline senza dati normalizzati.

## Decisione progettuale

1. **CML-085** sarà una slice runtime per mostrare in Curriculum lo stato di completezza per disciplina:
   - "Bozza completa disponibile" → solo Tecnologia
   - "Solo consultazione" → tutte le altre
   - "Struttura da completare" / "Non pronta per approvazione"

2. **CML-086** introdurrà stati di approvazione veri e propri, ma solo per discipline che hanno almeno il livello normalizzato completo.

3. Prima di normalizzare una nuova disciplina, servirà una slice di normalizzazione dedicata (es. CML-084A per Italiano, CML-084B per Matematica, ecc.).
