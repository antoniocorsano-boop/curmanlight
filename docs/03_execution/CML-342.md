# CML-342 — Home Onboarding Cue Solution Design

**Macroprogramma**: PM-06 — Accompagnamento
**Tipo slice**: docs-only / design-only
**Obiettivo**: Analizzare la Home attuale di CurManLight e proporre soluzioni concrete per il micro‑cue di onboarding, senza modificare runtime.

## Analisi della Home
- Screenshot desktop e mobile evidenziano una struttura a tre colonne con contenuti sovrapposti.
- Problemi di frizione: navigazione poco chiara per utenti scolastici non tecnici, assenza di elementi guidati.
- Utenti target: referenti curricolari, dipartimento, docente.

## Requisiti del micro‑cue
1. Evidenziare il primo passo di consultazione del curriculum.
2. Utilizzare linguaggio neutro (es. “proposta curricolare”).
3. Compatibilità con layout mobile‑first.
4. Non introdurre nuove dipendenze runtime.

## Alternative di design
| # | Descrizione | Pro | Contro |
|---|-------------|-----|-------|
| A | Banner statico in alto con icona “💡” e testo “Scopri la proposta curricolare”. | Visibile subito, zero JS. | Può occupare spazio verticale. |
| B | Tooltip interattivo al passaggio del mouse sul bottone “Curriculum”. | Guida contestuale, meno invasivo. | Richiede piccola logica JS (permesso runtime). |
| C | Card laterale “Inizia qui” con call‑to‑action. | Evidenzia passo chiave, good visual hierarchy. | Richiede redesign layout. |
| D | Micro‑animazione “pulse” su icona di onboarding. | Attira attenzione, minimo impatto. | Potrebbe distrarre alcuni utenti. |

## Matrice comparativa
- **Visibilità**: A > C > B > D
- **Impatto UI**: C > A > B > D
- **Complexità implementazione**: A (0) > D (minima) > B (leggera) > C (media)

## Raccomandazione
Adottare **Alternativa A** (Banner statico - `RECOMMENDED_ACTION_CALLOUT`) per la prima iterazione: garantisce visibilità immediata senza alcun cambiamento al runtime.
