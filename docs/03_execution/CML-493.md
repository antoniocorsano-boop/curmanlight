# CML-493 — Consultation State Model

## Obiettivo

Rendere espliciti e distinti gli stati della consultazione React, evitando che il caricamento asincrono venga mostrato come errore o assenza di dati.

## Problema osservato

`useCurriculum` restituiva solo `CurriculumDisciplina | null`. Il valore `null` rappresentava contemporaneamente:

- nessuna disciplina selezionata;
- caricamento in corso;
- modulo dati assente;
- errore di caricamento.

La UI non poteva quindi distinguere un normale caricamento da un problema reale.

## Modifiche

- introdotto `useCurriculumResource` con stati `idle`, `loading`, `success`, `error`;
- mantenuto `useCurriculum` per compatibilità con gli altri flussi;
- aggiunto comando `retry`;
- aggiunto stato di caricamento accessibile con `aria-live`;
- aggiunto stato di errore con azioni `Riprova` e `Cambia disciplina`;
- chiarito lo stato senza risultati filtrati;
- disabilitati i filtri dipendenti durante il caricamento;
- aggiornato il contratto automatico B02.

## Vincoli rispettati

- nessuna modifica ai dati curricolari;
- consultazione sempre read-only;
- nessuna azione decisionale;
- nessun backend o servizio remoto;
- persistenza e processo di revisione invariati.

## Validazione attesa

- React CI;
- B02 consultation contract;
- CML-470 Persistence Audit;
- CML-473 Comparative Audit;
