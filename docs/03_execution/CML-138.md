## Opzioni valutate

### Opzione A — Audit qualità contenuti disciplinari

Verificare completezza nodi per tutte le discipline.
**Vantaggio**: identifica gap contenutistici. **Svantaggio**: non risolve disallineamento architetturale.

### Opzione B — Selezione fonte dati canonica

Decidere se runtime deve usare dati hardcoded o JSON validati.
**Vantaggio**: risolve il problema strutturale alla radice.

**Vantaggio**: impatto immediato su utente. **Svantaggio**: non tocca il problema dati.

Verificare workflow umano post-modifiche.
**Vantaggio**: controllo di regressione. **Svantaggio**: non produce avanzamento core.

## Opzione selezionata

**Opzione B — Selezione fonte dati canonica**

Il disallineamento tra JSON disciplinari (7 file, validati, aggiornabili) e runtime (hardcoded MAPPA_DATI per sole 3 discipline, `TECNOLOGIA_NORM` per 1 sola) è il problema strutturale più grave. La scelta è: o si buttano i JSON e si continua con dati hardcoded, o si aggancia il runtime ai JSON come fonte unica. Raccomandazione: agganciare il runtime ai JSON.
