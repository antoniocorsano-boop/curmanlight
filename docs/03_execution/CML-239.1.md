# CML-239.1 — DEPLOY PAGES CONGRUENCE AND SYNTAX ERROR FIX

Runtime modificato: sì (PWA cache congruence)

## Obiettivo

Eliminare la non congruenza percepita tra HTML pubblicabile e HTML servito nel browser, causa principale delle anomalie `Unexpected identifier 'open'` e `setTab is not defined` in presenza di cache client stale.

- Branch: `main`

- Azione pre-obbligatoria eseguita: commit docs-only CML-239 separato (`0857038`)

## Diagnosi deploy congruence

1. Workflow Pages verificato: `.github/workflows/pages.yml` pubblica `"_published_snapshot/netlify-current"`.
2. Confronto byte/hash repository/live (canale terminale, senza service worker):
   - blob git `HEAD`/`origin/main` `_published_snapshot/netlify-current/index.html`: `826952` byte
   - live `https://antoniocorsano-boop.github.io/curmanlight/`: `826952` byte
   - hash blob coincidente (`d9ae7933cf1ed3338de548944ba7e6fab0d9ca16`)
3. In browser (tab già aperta con service worker attivo), `fetch('./index.html')` restituiva copia differente/stale (lunghezza testo inferiore), spiegando il mismatch osservato in audit CML-239.

Conclusione: il server Pages non stava pubblicando un file sbagliato; la divergenza era lato client per cache service worker.

## Diagnosi errori `open` / `setTab`

- Lo script live scaricato da terminale passa `node --check`.
- Nel browser con service worker attivo, una copia stale dell'HTML/script può portare a parse error (`open`) e quindi all'assenza delle funzioni inizializzate (`setTab`).
- `setTab` è presente nel file pubblicabile (`function setTab(...)`), quindi l'errore è conseguenza della mancata inizializzazione script, non della rimozione della funzione.

## Fix minimo applicato

### 1) `_published_snapshot/netlify-current/sw.js`

- Bump `CACHE_NAME` a `curmanlight-cache-v454-cml2391`.
- Strategia `network-first` per richieste di navigazione/HTML (`request.mode === 'navigate'` o `Accept: text/html`).
- Fallback a cache solo offline.

Effetto: evita il riuso persistente di HTML stale in presenza di nuove pubblicazioni.

### 2) `_published_snapshot/netlify-current/index.html`

- Registrazione SW aggiornata a:
  - `./sw.js?v=20260701-cml2391`
  - opzione `{ updateViaCache: "none" }`

Effetto: forza controllo aggiornamento del service worker senza riuso cache dell'update pipeline.
