# CML-486 — Cloudflare stale service worker reset

## Obiettivo

Impedire al service worker del runtime legacy, precedentemente pubblicato con scope `/`, di continuare a servire la vecchia interfaccia dopo il passaggio dell'applicazione React alla radice del dominio Cloudflare.

## Modifiche

- escluso `legacy/sw.js` dal pacchetto Cloudflare;
- generato un `sw.js` neutro alla radice che:
  - prende immediatamente il controllo;
  - elimina tutte le cache del dominio;
  - annulla la propria registrazione;
  - ricarica i client aperti;
- aggiunta una pulizia difensiva lato React delle registrazioni service worker e delle cache residue;
- mantenuti React su `/` e legacy su `/legacy/`.

## Invarianti

- nessuna modifica ai dati curricolari;
- nessuna modifica funzionale al runtime legacy;
- pubblicazione Cloudflare limitata a `_site`.

## Verdetto

`CML_486_CLOUDFLARE_STALE_SERVICE_WORKER_RESET_READY_FOR_REVIEW`
