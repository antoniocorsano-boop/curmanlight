Decidere quale micro-slice runtime aprire per prima dopo la fase di design, mock e visual contract.

CML-422 non implementa runtime. Definisce solo il perimetro della prima micro-slice ammissibile.

## Opzioni runtime valutate

### Opzione 1 — Impostazioni / Contesto di lavoro con anno scolastico

Descrizione:

```text
Introdurre o riallineare il Contesto di lavoro come funzione globale, includendo Anno scolastico, disciplina/ambito, ordine, classi assegnate e versione curricolare di riferimento.
```

Vantaggi:

```text
è fondazione per progettazione ed esportazione;
risolve anno scolastico;
abilita chip contesto in Home/header;
riduce ripetizioni nei documenti;
coerente con Visual Contract;
può essere micro-slice limitata.
```

Rischi:

```text
richiede verifica attenta del profilo utente già esistente in index.html;
non deve rompere salvataggio locale o export esistenti;
non deve introdurre dati personali sensibili.
```

Valutazione:

```text
PRIORITÀ MASSIMA
```

### Opzione 2 — Home D2 con chip contesto e quattro azioni guidate

Descrizione:

```text
Riallineare Home al modello D2: domanda guida, quattro accessi, chip contesto, validazione umana compatta.
```

Vantaggi:

```text
impatto visivo immediato;
allinea percezione utente;
facilita test live.
```

Rischi:

```text
senza Contesto di lavoro consolidato il chip sarebbe incompleto;
rischio UI senza fondazione dati.
```

Valutazione:

```text
SECONDARIA, dopo o insieme a contesto minimo
```

### Opzione 3 — Esportazione per documento con avviso contesto mancante

Descrizione:

```text
Riallineare Esportazione al bisogno scolastico: documento prima del formato, con avviso se il contesto manca.
```

Vantaggi:

```text
alto valore per docente;
coerente con CML-421B;
prepara programmazione, programma svolto, relazione finale.
```

Rischi:

```text
senza Contesto di lavoro completo potrebbe essere prematura;
rischio di aprire molti output nuovi insieme.
```

Valutazione:

```text
TERZA, dopo contesto e Home minima
```

## Decisione

La prima micro-slice runtime deve essere:

```text
CML-423 — Work Context and School Year Runtime Micro-slice
```

Perimetro:

```text
Impostazioni / Contesto di lavoro
Anno scolastico
chip contesto sintetico
salvataggio locale controllato
nessuna modifica profonda agli export
nessuna modifica curricolare
nessun deploy automatico
```

## Perché CML-423 prima delle altre

```text
Anno scolastico è dato globale.
Progettazione ed Esportazione dipendono dal Contesto di lavoro.
Home D2 richiede chip contesto.
I documenti scolastici per classe richiedono anno scolastico, disciplina e classi.
```

## Perimetro CML-423 ammesso

### Dentro

```text
leggere il profilo/contesto già presente in index.html;
identificare campi esistenti riusabili;
agggiungere/normalizzare anno scolastico nel contesto locale;
mostrare chip contesto sintetico se disponibile;
mostrare avviso non invasivo se anno scolastico manca;
aggiornare solo runtime pair se si implementa: index.html + _published_snapshot/netlify-current/index.html;
mantenere privacy locale;
mantenere validazione umana;
```

### Fuori

```text
nuovo backend;
login;
Drive/Workspace;
nuovi export completi;
refactor ampio;
modifica dati curricolari;
nuova navigazione completa;
implementazione intero mock D2;
deploy automatico;
merge su main senza controllo.
```

## Gate tecnico per CML-423

Prima di implementare CML-423 occorre:

```text
1. leggere index.html nelle aree profilo/contesto/salvataggio/export;
2. identificare le funzioni esistenti da riusare;
3. definire modello minimo di contesto;
4. preparare patch piccola su runtime pair;
5. eseguire diff-check;
6. eseguire validatori curriculum e shape test;
7. eseguire smoke visivo almeno Home + Impostazioni se possibile;
8. produrre report CML-423.
```

## Acceptance criteria CML-423

```text
Anno scolastico configurabile nel Contesto di lavoro.
Contesto salvato localmente.
Chip contesto mostra almeno disciplina/ordine/anno scolastico se disponibili.
Se anno scolastico manca, messaggio chiaro e non bloccante salvo export/documenti per classe.
Nessuna modifica ai dati curricolari.
Nessun cambio al flusso .cml esistente.
Runtime pair allineato.
```
