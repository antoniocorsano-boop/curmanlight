## Obiettivo

Produrre un piano completo di migrazione per sostituire il runtime legacy (`_published_snapshot/netlify-current/index.html`) con l'applicazione React (`curman-react/`), attraverso un inventario sistematico della parità funzionale, dei dati, operativa e dell'esperienza utente.

Non si tratta di riscrivere il runtime, ma di mappare con precisione ciò che serve prima che React possa diventare candidato alla sostituzione.

## Perimetro

L'intero sistema applicativo legacy:

- Home e orientamento operativo
- Curriculum: Consultazione, Revisione (con gap layer), Esportazioni, Processo
- Didattica: Evidenze e Valutazione, Programmazione Annuale, UDA Modello
- Persistenza locale e recupero dati
- Backup e ripristino
- Importazione/esportazione `.cml`
- PWA, service worker, cache
- Accessibilità, responsività, dispositivi mobili
- Messaggi, errori, stati vuoti

## Classificazione priorità

| Livello | Significato | Regola |
|---------|-------------|--------|
| **P0** | Indispensabile per sostituire il legacy | Bloccante per il passaggio |
| **P1** | Importante ma differibile | Non blocca solo con decisione formale |
| **P2** | Miglioramento nuovo o sperimentale | Fuori dalla migrazione di parità |

## Quattro tipi di parità

1. **Funzionale**: ogni azione legacy ha equivalente React con stessi prerequisiti, risultati, stati vuoti/errore
2. **Dati**: struttura curricolare, identificativi, decisioni, formati `.cml` — compatibilità preservata
3. **Operativa**: caricamento, salvataggio, recupero sessione, backup, import/export, offline, cache, ripristino
4. **Esperienza utente**: leggibilità, navigazione, responsività, accessibilità, messaggi, continuità contesto

## Aspetti critici

- **Persistenza**: legacy (localStorage) → React (Zustand + Dexie/IndexedDB) — evitare doppio stato concorrente
- **Compatibilità `.cml`**: prove bidirezionali legacy↔React su export/import
- **Service worker**: nuova strategia cache, eliminazione cache legacy, primo avvio dopo passaggio, procedura di rollback
- **Doppio runtime**: React non deve diventare seconda app produttiva prima del passaggio ufficiale
