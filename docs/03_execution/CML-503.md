# CML-503 — Referent Validation

## Obiettivo

Consentire al Referente di registrare localmente la validazione finale per ogni esito dipartimentale importato.

## Implementazione

- valori disponibili: `confermata`, `respinta`, `sospesa`;
- nota facoltativa per la conferma;
- nota obbligatoria per respinta e sospesa;
- persistenza locale in `curmanlight.referent-validations.v1`;
- associazione stabile tramite fingerprint del file e `proposalId`;
- modifica e rimozione della singola validazione;
- rimozione completa delle validazioni locali;
- riepilogo confermate, respinte, sospese e ancora da validare;
- accesso protetto dallo stesso codice operativo già richiesto per l’import Referente.

## Confine della slice

CML-503 registra lo stato finale del Referente ma non genera ancora il file `referent_validation.cml`. L’export finale resta separato nella slice successiva.

## Vincoli

- nessun backend;
- nessun upload;
- nessuna telemetria;
- nessun dato inviato all’esterno;
- validazione umana esplicita;
- persistenza esclusivamente nel browser locale.

## Validazione richiesta

- React CI;
- CML-470 Persistence Audit;
- CML-473 Comparative Audit;
- CML-491 Mobile Visual Audit;
- preview Cloudflare;
- review automatica;
- prova manuale dei tre stati, modifica, rimozione e reimport dello stesso file.

## Verdetto atteso

`CML_503_REFERENT_VALIDATION_READY_FOR_REMOTE_VALIDATION`
