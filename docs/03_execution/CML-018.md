## Razionale della semplificazione

Evitare strutture complesse, tecnicismi inutili e integrazioni premature. Il Drive condiviso deve essere un deposito unico senza organizzazione cartelle obbligatoria.

## Decisioni progettuali approvate

| #   | Decisione                                                          |
| --- | ------------------------------------------------------------------ |
| 1   | Unica cartella Drive condivisa senza sottocartelle obbligatorie    |
| 2   | File unico `.cml` come formato di scambio                          |
| 3   | Tre modalità di lavoro: Docente, Dipartimento, Referente curricolo |
| 4   | Automazione solo per invio proposta docente al Drive               |
| 5   | Import manuale guidato per Dipartimento e Referente                |
| 6   | Nessuna integrazione Drive bidirezionale nella prima versione      |
| 7   | Fallback manuale sempre disponibile                                |
| 8   | Validazione umana esplicita sempre richiesta                       |
