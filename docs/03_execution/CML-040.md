## Schema `.cml` utilizzato

Il formato `.cml` e JSON, schema `1.0`. Due tipi di file:

| Tipo                 | `fileType`           | Ruolo                                |
| -------------------- | -------------------- | ------------------------------------ |
| Proposta docente     | `teacher_proposal`   | Docente esporta proposte di modifica |
| Esito dipartimentale | `department_outcome` | Dipartimento restituisce esiti       |

### Campi richiesti (validazione app)

**teacher_proposal:**

- `fileType`: deve essere `"teacher_proposal"`
- `discipline`: stringa non vuota
- `counts`: oggetto
- `proposals`: array
- `humanValidationRequired`: `true`

**department_outcome:**

- `fileType`: deve essere `"department_outcome"`
- `discipline`: stringa non vuota
- `disciplineWorkStatus`: stringa non vuota
- `proposalHandling`: array
- `checks`: oggetto
- `humanValidationRequired`: `true`

### Valori ammessi per `handling`

| Valore (underscore)            | Display (UI dipartimento)      |
| ------------------------------ | ------------------------------ |
| `confluita_nella_sintesi`      | Confluita nella sintesi        |
| `riformulata_dal_dipartimento` | Riformulata dal dipartimento   |
| `assorbita_in_altra_proposta`  | Assorbita in altra proposta    |
| `da_chiarire`                  | Da chiarire                    |
| `senza_esito`                  | (fallback se non riconosciuto) |
