## Scopo

Preparare la prima micro-slice runtime effettiva:

```text
Contesto di lavoro + Anno scolastico
```

Questa slice documenta il contratto patch dopo lettura reale di `index.html`.

## Nota operativa

Il connettore remoto usato qui permette la sostituzione completa dei file, non una patch parziale sicura su file HTML molto grande.

Per evitare una modifica runtime fragile, questa slice chiude il contratto operativo e rimanda la patch effettiva a esecuzione locale o a strumento patch-aware.

## Decisione tecnica

La patch minima deve estendere `DEFAULT_PROFILE` senza rompere retrocompatibilità:

```js
const DEFAULT_PROFILE={
  nome:"",
  cognome:"",
  disciplina:DISCIPLINE[0],
  ordine:"Secondaria",
  annoScolastico:""
};
```

Perché è retrocompatibile:

```js
userProfile={...DEFAULT_PROFILE,...payload.profile}
```

aggiunge automaticamente il campo mancante ai salvataggi precedenti.

## Modello minimo CML-423

```text
userProfile.annoScolastico
```

```text

```


## Funzioni helper da introdurre

### getSchoolYearLabel

```js
function getSchoolYearLabel(){
  var value=(userProfile&&userProfile.annoScolastico?String(userProfile.annoScolastico).trim():"");
  return value ? "A.S. "+value : "Anno scolastico non impostato";
}
```

### getWorkContextChip

```js
function getWorkContextChip(){
  var parts=[];
  if(userProfile.disciplina) parts.push(userProfile.disciplina);
  if(userProfile.ordine) parts.push(ORDER_PROFILE_LABELS[userProfile.ordine]||userProfile.ordine);
  if(userProfile.annoScolastico) parts.push("A.S. "+userProfile.annoScolastico);
  return parts.join(" · ");
}
```

### requiresSchoolYearMessage

```js
function requiresSchoolYearMessage(){
  return "Completa l'Anno scolastico in Impostazioni / Contesto di lavoro per generare documenti per classe o report.";
}
```
