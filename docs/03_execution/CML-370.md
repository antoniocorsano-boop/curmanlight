# CML-370 — Discipline Data Access Contract and Centralization Plan

## Mandatory Slice Header
- Macroprogramma: (autonomo — docs-only contract design)
- Dipende da: CML-369 (external provider readiness audit)
- Tipo slice: docs-only contract
- Perimetro: nessuna modifica a runtime, curriculum, tools, assets, manifest, service-worker

## Obiettivo
Definire un contratto di accesso centralizzato ai dati disciplinari, eliminando i pattern fragili emersi in CML-369 (eccezione Tecnologia duplicata, 14-way if/else catena MAPPA_DATI, doppia nomenclatura). Solo contratto — nessuna implementazione runtime in questa slice.

## Risultati Ricognizione

### Pattern attuali frammentati

| Pattern | Localizzazioni | Problema |
|---|---|---|
| `discKey === 'tecnologia'` guard | 4 funzioni (L2962, L3085, L3287, L6362) | Hardcoded, fragile, Tecnologia favorita |
| `TECNOLOGIA_NORM_DATA` costante inline | L6202-L6204, ~3.5 KB | Duplicata dentro `ALL_CURRICULUM_DATA.tecnologia` |
| `ALL_CURRICULUM_DATA[discKey]` accesso diretto | 4 funzioni | Nessun fallback né validazione |
| 14-variabili `*_MAPPA_DATI` | L2711-L2846 | Catena if/else lunga 14 rami in `renderMappaDisciplinare()` |
| `discKeyFromName()` / `selDisc` doppia nomenclatura | 9 chiamate | Display name vs kebab-case, nessuna funzione inversa |

### Tre sorgenti dati distinti

1. **`DATA[displayName]`** — runtime editing state (traguardi/obiettivi con status)
2. **`ALL_CURRICULUM_DATA[kebabKey]`** — structured learning units (14 discipline)
3. **`*_MAPPA_DATI[kebabKey]`** — structural maps (14 variabili, accesso via if/else)

Solo i punti 2 e 3 sono oggetto di questo contratto. `DATA` ha uno scopo diverso (stato editing) e rimane separato.

## Contratto — DisciplineDataAccess

### 1. `getUnitsForDiscipline(discKey)` — Accesso unità apprendimento

Sostituisce: `getDisciplineEvidenceData()`, `getAnnualDraftUnitsFor()`, `getUdaDisciplineUnits()`, branch Tecnologia in `renderTecnologiaNorm()`

```
getUnitsForDiscipline(discKey) → UnitaApprendimento[] | []
```
- discKey: stringa kebab-case (es. "tecnologia", "arte-immagine")
- Cerca in `ALL_CURRICULUM_DATA[discKey].unitaApprendimento`
- Nessun branch speciale per Tecnologia
- Se discKey non trovato → `[]` (silent fallback)
- **Non accede mai a `TECNOLOGIA_NORM_DATA`**

### 2. `getMappaForDiscipline(discKey)` — Accesso mappa disciplinare

Sostituisce: 14-way if/else in `renderMappaDisciplinare()`

```
getMappaForDiscipline(discKey) → MappaDati | null
```
- discKey: stringa kebab-case
- Cerca in oggetto aggregato `MAPPA_DATI_INDEX[discKey]`
- Se discKey non trovato → `null`
- **Elimina la catena if/else di 14 rami**

### 3. `getCurriculumDataForDiscipline(discKey)` — Accesso record completo

```
getCurriculumDataForDiscipline(discKey) → CurriculumRecord | null
```
- Restituisce il record completo da `ALL_CURRICULUM_DATA[discKey]`
- Se discKey non trovato → `null`

### 4. Funzioni di mappa

```
discKeyToName(discKey) → string | null   // inverso di discKeyFromName
discKeyFromName(name) → string | null    // già esistente
```
- `discKeyToName()` è il reverse di `discKeyFromName()` — mancante, va aggiunto

### 5. Validazione e error handling

```
isValidDiscKey(discKey) → boolean         // esiste in ALL_CURRICULUM_DATA
listAllDiscKeys() → string[]              // tutte le chiavi disponibili
listAllDiscNames() → string[]             // tutti i nomi display (≡ DISCIPLINE)
```

### 6. Eliminazione dati duplicati

- `TECNOLOGIA_NORM` e `TECNOLOGIA_NORM_DATA` (L6202-L6204) **vanno rimossi**
- Il contenuto rimane in `ALL_CURRICULUM_DATA.tecnologia` (unica fonte)
- `schemaVersion`, `humanValidationRequired`, `source` (campi extra di TECNOLOGIA_NORM) persi? Valutare se servono — sembrano metadati di prototipo non usati dal runtime

### 7. MAPPA_DATI aggregazione

- Le 14 variabili `*_MAPPA_DATI` (L2711-L2846) **vanno aggregate** in un unico oggetto `MAPPA_DATI_INDEX`
- Chiavi: kebab-case (discKey)
- Accesso: `getMappaForDiscipline(discKey)`
- La variabile singola per disciplina non serve più
