## Decisione di roadmap

**Opzione selezionata**: **B** — Prossima slice runtime minima su UX-001.

**Motivazione**:
1. UX-001 è P0 e blocca l'avanzamento di PM-04 oltre il 55%.
2. CML-293 (semplificazione testata disciplina) è già pianificato nella roadmap con dipendenza CML-283.
3. Un intervento mirato sulla testata disciplina risolve sia UX-001 (logiche miste) sia UX-011 (testata sovraccarica) in un'unica slice.
4. Le opzioni A (docs-only) ed E (mantenimento) non produrrebbero avanzamento misurabile. L'opzione C (UX-009) è priorità P2. L'opzione D (chiusura PM-04) è prematura senza runtime.

**Prossima slice consigliata**: CML-293 — Semplificazione testata disciplina.
**Dopo CML-293**: PM-04 potrà essere aggiornato a ~70%.
