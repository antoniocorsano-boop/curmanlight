## Formal Multi-Discipline Package Format

**Non introdotto.** CML-206 ha selezionato Option A: mantenere i tipi di file v1.0 esistenti e l'import batch multiplo. Nessun nuovo `fileType`, nessun bump schema, nessuna complessità aggiuntiva.

## Deferred UX Options

| Opzione | Motivo Defer |
|---|---|
| Preview panel before import | P1 ma richiede nuovo UI state, thumbnail/summary generation, cancel flow; out of scope per minimal increment |
| Drag-and-drop | P3; aggiunge complessità (drag events, drop zones, mobile touch) senza forte domanda utente |
| Reset/clear import button | P3; utente può ricaricare pagina; bassa urgenza |
| Richer recovery guidance per error code | P2; messaggi attuali sono sufficienti; miglioramento iterativo futuro |
| Schema version bump per packages | Non necessario; rejected by CML-206 |

## Perché i Deferred UX Non Bloccano la Closure

1. **Preview panel**: il batch import funziona; l'utente vede i risultati post-import con warnings/chips. La preview pre-import è un miglioramento nice-to-have, non un blocker operativo.
2. **Drag-and-drop**: il file picker multiplo funziona su desktop e mobile. Il drop-zone aggiunge complessità senza evidenza di dolore utente.
3. **Reset button**: reload pagina è accettabile in un'app statica senza stato persistente.
4. **Richer recovery guidance**: i messaggi attuali (`Mancano: X`, `Senza testo`, `Versione schema non indicata`) sono già sufficienti per guidare l'utente.
