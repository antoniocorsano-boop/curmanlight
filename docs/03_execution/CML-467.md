# CML-467 — B03 Decision Workflow Completion

## Scopo

Completare in un'unica slice il flusso decisionale B03 rimasto aperto dopo CML-463–466, riducendo i cicli ripetitivi di integrazione senza sacrificare i controlli.

## Implementazione

- idratazione automatica delle `WorkDecision` all'avvio React;
- stato esplicito della persistenza: idle, loading, restored, saved, error;
- feedback accessibile nella vista Revisione;
- azione `revision_requested` con motivazione obbligatoria;
- azione `accepted_custom` con testo personalizzato e nota facoltativa;
- visualizzazione coerente degli esiti e riapertura;
- proiezione legacy limitata agli esiti finali compatibili;
- salvataggio coordinato anche per undo e reset;
- ampliamento del contratto automatico B03.

## Compatibilità

- nessuna modifica al runtime legacy pubblicato;
- nessun backend o invio dati;
- persistenza esclusivamente locale;
- API legacy dello store mantenute temporaneamente;
- `revision_requested` resta non finale e non viene trasformata in approvazione legacy.

## Controlli

- React CI completa;
- lint;
- B01/B02/B03;
- TypeScript;
- build Vite;
- validazione locale finale unica prima del merge.
