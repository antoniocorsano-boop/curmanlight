# CML-081 — CURRICULUM AREA COMPLETION RUNTIME INCREMENT

## Objective

Make the Curriculum area fully implementato e impostato per la revisione e validazione: mappatura visiva completa dell'intero curricolo, guida esplicita al percorso consulta→revisione→decisione.

## Current gap

- `tab-curricolo` (Consulta) mostra discipline e item ma non dà visibilità immediata sulla completezza del curricolo
- Nessun indicatore di stato per-disciplina (quanti item, quanti pending, quanti approvati)
- Nessuna guida esplicita al percorso: l'utente atterra su Consulta e deve sapere di andare su Revisione
- La mappa completa del curricolo (14 discipline × 3 ordini) non è evidente

## Changes

All changes in `renderCurricoloIstituto()` function in `index.html`:

### 1. Curriculum completeness banner (top of view)

- Show total count: 14 discipline, N items total across Infanzia/Primaria/Secondaria
- Badge: "Mappa completa del curricolo di istituto"

### 2. Path guidance box

- Replace existing `usage-notice` with more explicit guidance:
  - "📖 Consulta: leggi il curricolo vigente e confronta con la proposta 2025"
  - "✏️ Revisione: valuta ogni voce e decidi se approvare, modificare o discutere"
  - "📋 Definitivo: riepilogo delle decisioni prese"
  - "📤 Esporta: genera il documento per il Collegio Docenti"

### 3. Per-discipline status summary section

- Between the index pills and the first ordine section, add a compact summary grid:
  - Per each discipline: icon, name, total items, count of pending decisions
  - Color coding: green (all decided), yellow (some pending), red (many pending)

### 4. "Next action" callout in each discipline section

- After each discipline's items, add: "✏️ Vuoi revisionare questa disciplina? Vai alla scheda Revisione"
- Link to `setTab('lavoro')` for quick navigation

### 5. Ordnung section visibility

- Ensure each ordine section shows a count of disciplines and items covered

## Files modified

- `_published_snapshot/netlify-current/index.html` — only `renderCurricoloIstituto()` function

## Constraints

- No new JS functions or data structures
- No changes to `.cml` schema, export/import, role-access
- No external dependencies
- No changes to other tabs or views
- CSS variables from CML-078 used for consistency
