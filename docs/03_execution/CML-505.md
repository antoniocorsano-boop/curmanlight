# CML-505 — End-to-End Workflow Audit

Baseline: `23ff827fea0f5ca2e216178dc7fa7d0433953e3c`.

Percorso verificato:

`teacher_proposal.cml → department_outcome.cml → referent_validation.cml`

## Esito

Il flusso non è ancora chiudibile end-to-end.

### P1 — export Docente incompatibile con import Dipartimento

Il builder React della proposta docente non valorizza `motivazione` e `fonte`. Il validatore dell’import Dipartimento richiede entrambe come stringhe non vuote. Un file prodotto dal Docente React può quindi essere respinto dal Dipartimento React.

Correzione richiesta: aggiungere i due campi a `ProposalItem` e derivarli da `GapEntry.motivazione` e `GapEntry.sourceRefs`, con fallback espliciti.

### P1 — parser comune incompleto

Il tipo `CmlFile` comprende `referent_validation`, ma `parseCmlFile` gestisce soltanto `teacher_proposal` e `department_outcome`. L’export finale del Referente non è quindi riconosciuto dal parser comune.

Correzione richiesta: aggiungere il caso `referent_validation` e una matrice di test positiva e negativa per tutti e tre i formati.

## Controlli superati

- import multiplo e deduplicazione su file conformi;
- coda e decisioni dipartimentali persistenti;
- import e validazione Referente protetti dal codice operativo;
- fallback legacy per ordine e anno scolastico;
- tracciabilità tramite fingerprint e `proposalId`;
- nessun backend, upload o telemetria.
