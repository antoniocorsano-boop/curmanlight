# CML-436 — Pages Stale Home Cache Invalidation and Content Smoke

- Macroprogramma: PM-09 Validazione con utenti (infrastruttura deploy)
- Dipende da: CML-435 (Home Mock Conformance Visibility Gate)
- Tipo slice: runtime micro-fix (cache/service worker)
- Runtime: modificato (index.html, _published_snapshot/netlify-current/index.html, sw.js)
- Branch: codex/cml-436-pages-stale-home-cache-invalidation (pushata su origin, non mergiata)

## Obiettivo

Forzare GitHub Pages a servire la Home CML-435 reale invalidando la cache del service worker. CML-435 era mergiata correttamente ma la cache del service worker serviva contenuto stale.

## Causa

- Service worker cache name: `curmanlight-cache-v454-cml2391` (non cambiato da CML-2391)
- `APP_SHELL` include `./index.html` — versione vecchia cached su install
- Anche con network-first per HTML, il vecchio SW rimaneva attivo e non rilevava il nuovo bundle come "diverso" perché la registration URL `sw.js?v=20260701-cml2391` non cambiava
- `updateViaCache: "none"` non bastava: il browser confronta byte del SW, ma se il SW stesso non cambiava abbastanza da triggerare update, l'install non partiva

## Intervento

1. **`sw.js`**: `CACHE_NAME` bumpato da `v454-cml2391` a `v455-cml436` — forza purge cache vecchia su activate
2. **`index.html` (entrambi)**: registration URL da `./sw.js?v=20260701-cml2391` a `./sw.js?v=20260710-cml436` — forza browser a riconoscere nuovo SW
3. **`sw.js` copiato in root**: per allineamento dev locale

## Verifica live

```bash
curl -L "https://antoniocorsano-boop.github.io/curmanlight/?cml436=$(date +%s)" | grep -c "Apri il curricolo"
```
