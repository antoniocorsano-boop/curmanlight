# CML-484 — Cloudflare Pages continuous deployment readiness

## Obiettivo

Predisporre CurManLight per una pubblicazione secondaria gratuita su Cloudflare Pages, collegata al repository GitHub.

## Architettura di pubblicazione

- branch di produzione: `main`;
- comando di build: `bash scripts/build-cloudflare-pages.sh`;
- directory di output: `_site`;
- Node.js: 20;
- runtime legacy pubblicato alla radice;
- applicazione React pubblicata in `/react-preview/`;
- nessun backend, funzione serverless, telemetria o servizio remoto applicativo.

## Funzionamento

Lo script:

1. ricrea `_site`;
2. copia `_published_snapshot/netlify-current/` nella radice dell'artefatto;
3. installa le dipendenze React con `npm ci`;
4. esegue la build Vite con `VITE_BASE_PATH=/react-preview/`;
5. copia `curman-react/dist/` in `_site/react-preview/`.

## Configurazione account-side Cloudflare

Nel pannello Cloudflare Pages:

- collega `antoniocorsano-boop/curmanlight` tramite GitHub;
- usa `main` come production branch;
- lascia vuota la root directory;
- usa `bash scripts/build-cloudflare-pages.sh` come build command;
- usa `_site` come build output directory;
- imposta `NODE_VERSION=20` se richiesto;
- abilita preview deployment per le pull request.

Cloudflare Pages genera automaticamente preview URL per branch e pull request quando il progetto usa la Git integration.

## Stato

Repository pronto. Il collegamento GitHub → Cloudflare richiede autorizzazione interattiva nel pannello Cloudflare.
