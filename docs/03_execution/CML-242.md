# CML-242 — HERMES ASSISTED MODE GOVERNANCE CONTRACT

## Overview
This document defines the operational contract for using Hermes in assisted mode on the CurManLight project.

## Key Rules
1. **No Pushes** – Hermes must never push commits to any remote repository.
2. **Pre‑commit disclosure** – Before any commit Hermes must show:
   - `git status --short --branch`
   - List of modified files
   - Summary of the diff
   - Proposed commit message
3. **Docs‑only slices** are allowed only in:
   - `docs/03_execution/`
   - `report/`
   - `docs/REPO-MOVELOG.md`
4. **Runtime slices** require a dedicated slice and explicit approval.
5. **Deploy slices** require separate explicit approval.
6. **README.md** modifications are prohibited unless an explicit request is made.
7. **Minimal secret scan** is performed on each operation; any credential leak aborts the operation.
8. **Verdict** – `CML_242_HERMES_ASSISTED_MODE_GOVERNANCE_CONTRACT_READY_LOCAL_NOT_PUSHED` – the commit stays local until manual push authorization.

9. **Hermes must not commit after a clarification timeout.**

*Signed off – Hermes Governance*
