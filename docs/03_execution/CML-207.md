## Obiettivo

Audit e definizione di un contratto UX minimale per il workflow di import batch `.cml` esistente, senza modifiche a schema, validatori o export/import behavior.

## Current UX Findings

### 4.1 Teacher Proposal Import (Department UI)

- **Entry point**: bottone "Importa proposte .cml" con `<input type="file" multiple accept=".cml">`
- **Pre-importer feedback**: "Seleziona almeno un file .cml." se nessun file selezionato
- **Post-importer feedback**: toast "✅ Proposte docenti importate localmente" o "⚠️ Nessun file .cml valido importato"
- **File list rendering**: ogni file mostrato come card con `valid`/`invalid` class, nome file, messaggio, e extra (duplicato, warnings, missing fields)
- **Warnings rendering**: `file.warnings.join("; ")` mostrati inline nella card del file
- **Mixed-discipline warning**: chip informativo "ℹ️ Sono presenti discipline diverse: il dato è segnalato ma non blocca la lettura."
- **Duplicate warning**: chip "Duplicato probabile" nella card del file e come issue nella proposta
- **Proposal grouping**: raggruppamento per `discipline` → `status` con sezioni `<section>` e sottotitoli `<div>`
- **Missing fields**: elencati come "Mancano: discipline, counts, ..."
- **Preview before import**: none — il file è parsato e importato direttamente
- **Recovery guidance**: none esplicita oltre ai messaggi di errore

### 4.2 Department Outcome Import (Referent UI)

- **Entry point**: bottone con role-access gate ("Codice operativo richiesto") + `<input type="file" multiple accept=".cml">`
- **Pre-importer feedback**: "Seleziona almeno un file .cml." se nessun file selezionato
- **Post-importer feedback**: toast "✅ Esiti dipartimentali importati localmente" o "⚠️ Nessun file .cml valido importato"
- **File list rendering**: lista di file con `valid`/`invalid` class, nome, messaggio, warnings, missing
- **Mixed-discipline warning**: stesso chip informativo del department import
- **Duplicate warning**: flaggato ma non visivamente distinto nel file list (manca il rendering esplicito del chip "Duplicato probabile" nel referente, a differenza del department)
- **Outcome rendering**: card per outcome con header (discipline, status), dettagli (proposte totali, confluita, riformulata, assorbita, da chiarire, senza esito)
- **Preview before import**: none
- **Recovery guidance**: none esplicita

## UX Gaps by Severity

### P1 — High Impact

| # | Gap | Impact | Current Behavior |
|---|---|---|---|
| 1 | No preview before import | User imports files blindly; cannot verify content before commit | Files parsed and imported immediately on selection |
| 2 | Duplicate rendering inconsistency | Referent import does not show "Duplicato probabile" chip visually | Department shows chip; referent stores flag but doesn't render it |
| 3 | Mixed-discipline message generic | User doesn't know *which* disciplines are mixed | Chip says "Sono presenti discipline diverse" but doesn't list them |

### P2 — Medium Impact

| # | Gap | Impact | Current Behavior |
|---|---|---|---|
| 4 | No import summary before/after | User lacks quick "what happened" overview | Summary stats exist but are buried in card layout |
| 5 | No recovery guidance per error code | User must infer next action from generic messages | "Mancano: X, Y" but no "aggiungi X e Y al file e riprova" |
| 6 | File size warnings not prominent | Large files cause slow UI but warning is inline | Warning is in `file.warnings` array, rendered as text |

### P3 — Low Impact

| # | Gap | Impact | Current Behavior |
|---|---|---|---|
| 7 | No drag-and-drop | Users must click file picker | Standard `<input type="file">` only |
| 8 | No file count limit | User could select 100+ files | No UI feedback on batch size |
| 9 | No "clear all" / "reset import" | User must reload page to start over | No reset button visible |

## Selected Minimal UX Increment for CML-208

The audit selects **three focused UX improvements** for CML-208, keeping changes minimal and local to the import UI:

### 1. Clearer Multi-File Selection Labels (P1)
- Add helper text near the file input: "Seleziona uno o più file .cml. Puoi selezionare più file contemporaneamente."
- This clarifies multi-file intent without changing behavior.

### 2. Mixed-Discipline List Warning (P1)
- Enhance the mixed-discipline chip to list the actual disciplines present: "Sono presenti discipline diverse: [Tecnologia, Italiano]. Il dato è segnalato ma non blocca la lettura."
- Uses `summary.disciplines` already available in `buildDepartmentImportSummary`.

### 3. Duplicate Rendering Consistency (P1)
- Add "Duplicato probabile" chip rendering in the referent file list, matching the department behavior.
- Uses `file.duplicate` flag already stored in `referentOutcomeState.files`.

### 4. Import Summary Banner (P2)
- After import, show a compact banner: "Importati N file: M validi, K non riconoscibili, X da controllare."
- Uses existing `summary` data; no new computation.

### What CML-208 Will NOT Include

- No drag-and-drop (deferred — P3, not justified by current evidence)
- No preview before import (deferred — requires significant UI work, P1 but low urgency given batch import works)
- No schema changes
- No validator changes
- No export/import behavior changes
