## Metodo di audit

1. **Caricamento live** della pagina `https://antoniocorsano-boop.github.io/curmanlight/`
2. **Analisi statica** del DOM (322KB) su 11 criteri:
   - Architettura informativa
   - Navigazione
   - Gerarchia visiva
   - Densità azioni
   - Separazione Curriculum/Didattica
   - Mobile usability
   - Accessibilità base
   - Microcopy
   - Design system leggero
   - Sostenibilità futura
3. **Classificazione** problemi: P0 (bloccante), P1 (confusione maggiore), P2 (sovraccarico/incoerenza), P3 (rifinitura)
4. **Raccomandazione** prossimo step

## Cosa funziona bene

1. **Home due aree**: la separazione Curriculum/Didattica è chiara, le card sono ben dimensionate
2. **Microcopy prudenziale**: "Non costituisce documento istituzionale", "Da adattare alla classe" sono efficaci
3. **Separazione istituzionale/operativo**: il modello concettuale è solido
4. **UDA read-only**: 2 modelli con badge origine dati 🟢/🟡 trasparenti
5. **Valutazione/Evidenze**: filtro ordine, card espandibili, dati collegati al curricolo
6. **Version switcher 2012/2025**: chiaro e immediato
7. **Mobile bottom bar**: 5 pulsanti, touch target adeguati
8. **Role-access gating**: codice operativo semplice, microcopy onesta
