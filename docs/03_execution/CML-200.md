## Regole contratto implementate

### 1. Enforcement `schemaVersion`
- `schemaVersion` presente e diverso da `"1.0"` → bloccato con codice `unsupported_schema`
- `schemaVersion` non-stringa → bloccato come `unsupported_schema`
- `schemaVersion` mancante → accettato con warning `legacy_version` ("Versione schema non indicata; accettato come compatibile.")

### 2. Codici di validazione strutturati
Aggiunto campo `warnings: []` ai risultati di validazione:
- Blocco: `not_recognizable`, `invalid_json`, `wrong_type`, `missing_data`, `unsupported_schema`, `too_large`, `duplicate_handling`
- Warning: `legacy_version`, `missing_semantic_field`, `duplicate`, `mixed_discipline`, `discipline_mismatch`
- Informativo: messaggi di successo, assenza endpoint Drive

### 3. Validazione `teacher_proposal`
- Top-level required: `schemaVersion`, `fileType`, `discipline`, `counts` (oggetto), `proposals` (array), `humanValidationRequired` (true)
- Item-level warnings: `id`, `proposta`, `motivazione`, `fonte` vuoti → chip "Senza id", "Senza testo", "Senza motivazione", "Senza fonte"
- Troncamento campi stringa > 100.000 caratteri con warning

### 4. Validazione `department_outcome`
- Top-level required: `schemaVersion`, `fileType`, `discipline`, `disciplineWorkStatus`, `proposalHandling` (array), `checks` (oggetto), `humanValidationRequired` (true)
- `discipline`/`disciplines` consistency: se `disciplines` presente e non nullo, warning se `disciplines[0]` non coincide con `discipline`
- Duplicate `proposalId` nello stesso file → bloccato con codice `duplicate_handling`
- Handling vuoto/null → accettato come `senza_esito` (comportamento esistente preservato)

### 5. Duplicate detection
- File duplicati: fingerprint basato su `discipline`, `createdAt`, contenuto righe → warning "Duplicato probabile", incluso nello stato
- Proposal ID duplicati: warning per-item, incluso nello stato
- Handling ID duplicati in stesso file: bloccato

### 6. Size/array limits
- File > 20 MB: bloccato con `too_large`
- File > 5 MB: warning prestazioni
- Array `proposals[]` o `proposalHandling[]` > 2000: bloccato con `too_large`
- Array > 500: warning prestazioni

### 7. Preservazione campi sconosciuti
- Nessun campo viene strippato dal modello parsato
- Campi extra conservati e ignorati durante l'elaborazione

### 8. Messaggi utente migliorati
- `not_recognizable` / `wrong_type`: "Questo file non è una [tipo] CurManLight."
- `unsupported_schema`: "Versione schema non supportata. Aggiorna CurManLight o esporta di nuovo il file."
- `invalid_json`: "Il file contiene JSON non valido."
- `missing_data`: "Il file è leggibile, ma mancano alcune informazioni." + elenco campi mancanti
- `duplicate`: chip "Duplicato probabile"
- `legacy_version`: chip "Versione schema non indicata; accettato come compatibile."
- `too_large`: "File troppo grande per l'importazione locale. (max 20 MB)"
