- Nessuna modifica al runtime.
- Nessuna modifica a `_published_snapshot/netlify-current/index.html`, salvo blocker critici da documentare.
- Nessuna modifica a PDF, `sw.js`, `_headers`, asset.
- Nessun deploy.
- Nessuna introduzione di Drive API, Apps Script, endpoint reali, autenticazione.
- Nessuna modifica allo schema `.cml`.
- Nessuna reintroduzione della semantica rigida `accolta / non_accolta`.
- Nessuna produzione ancora del report finale per il gruppo di lavoro (out of scope).

**Flusso verificato:**

1. **Docente:** esporta proposta `.cml` (`fileType: teacher_proposal`).
2. **Dipartimento:** importa la proposta, assegna un esito del confronto (con le quattro opzioni collegiali), esporta esito dipartimentale (`.cml` con `fileType: department_outcome`).
3. **Referente:** importa l’esito dipartimentale, visualizza riepilogo con discipline, esiti totali, conteggi `da_chiarire` e `senza esito`.

**Verifiche eseguite (tutte PASS):**

- **Apertura app:** nessun errore JavaScript; tab `Revisione` raggiungibile; aree docente, dipartimento e referente presenti e coerenti.
- **Proposta docente `.cml`:** generata tramite `exportTeacherCml()`; JSON valido; contiene `fileType: teacher_proposal`, `humanValidationRequired: true`, `discipline`, `counts`, `proposals`, `checks`.
- **Invio Drive predisposto:** pulsante `Invia al Drive condiviso` mantiene comportamento endpoint non configurato (alert se mancante); nessuna chiamata di rete automatica; fallback `Scarica proposta .cml` sempre disponibile.
- **Import dipartimento:** importa la proposta docente; riconosciuta come valida; mostra file importati, proposte, disciplina, controlli e criticità.
- **Gestione esito del confronto:** assegnabili tutti e quattro gli esiti (`confluita_nella_sintesi`, `riformulata_dal_dipartimento`, `assorbita_in_altra_proposta`, `da_chiarire`); etichette utente collegiali; assenza di vecchie etichette rigide (`accolta`, `accolta con modifiche`, `non accolta`, `respinta`).
- **Export esito dipartimento `.cml`:** generato tramite `exportDepartmentOutcomeCml()`; JSON valido; contiene `fileType: department_outcome`, `humanValidationRequired: true`, `disciplineWorkStatus: "completed"`, `proposalHandling[]`, `checks.hasHandling`.
- **Import referente:** importa l’esito dipartimentale; riconosciuto come valido; pannello `Verifica referente curricolo` mostra file importati, file validi, discipline rilevate, esiti totali, elementi `Da chiarire`, elementi `Senza esito`, eventuale segnalazione discipline miste.
- **Gestione `da_chiarire`:** proposte con `handling: "da_chiarire"` conteggiate e mostrate; non bloccano il flusso; evidenziate come punti da portare all’attenzione.
- **Gestione senza esito:** handling vuoto o mancante mostrato come `Senza esito del confronto`; non trasformato automaticamente in `da_chiarire`; non nascosto.
- **File non validi:** rifiutati e segnalati chiaramente in ciascuna fase (non `.cml`, JSON invalido, `fileType` errato, schema parziale).
- **Rendering sicuro:** tutti i dati importati da file `.cml` passati da `escapeHtml` prima dell’inserimento in `innerHTML`; nessun payload HTML/script eseguito.
- **Assenza rete/persistenza:** nessuna chiamata di rete durante le operazioni di download/import/export locale; nessuna nuova persistenza permanente; nessun uso di Google API, Apps Script, endpoint reale, autenticazione.
- **Regressioni:** riepilogo/export senza marker tecnici `[IN 2025: ...]`; indicatore ✏️ coerente; download docente `.cml` funzionante; fallback Drive funzionante; import docente funzionante; export dipartimento funzionante; import referente funzionante.
- **Responsive:** le aree risultano leggibili e usabili a larghezze 390 px, 768 px, 1280 px (layout basato su CSS esistente, nessuna modifica in questa slice).
- **Confini repository:** PDF, `sw.js`, `_headers`, asset invariati; nessun deploy eseguito.

**Esiti principali:**

- Tutto il flusso locale Docente → Dipartimento → Referente funziona senza interruzioni.
- I dati vengono trasferiti correttamente tra le fasi tramite file `.cml`.
- Le etichette utente sono interamente collegiali e rispettano il principio di validazione umana e collegiale.
- Nessuna regressione introdotta nelle funzionalità esistenti.
