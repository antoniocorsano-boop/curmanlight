# CML-482 — Version Registry Data Model and Validator

## Obiettivo

Tradurre il contratto CML-481 in un modello dati eseguibile e in un validatore automatico, senza registrare ancora il dataset curricolare corrente e senza attribuirgli un quadro normativo non documentato.

## Baseline

- `main`: `c1dd3283a1b4a57a97160a3ba3fafa81275848ec`
- contratto: `docs/02_system/DUAL-CURRICULUM-VERSION-REGISTRY-CONTRACT.md`

## Modello dati

File:

```text
curman-react/src/types/curriculum-version.ts
```

Definisce:

- framework `IN_2012`, `IN_2025`, `UNSPECIFIED`;
- scope `nazionale`, `istituto`, `repository`;
- ciclo di vita documentale;
- stato deliberativo;
- identità, provenienza, intervallo temporale e relazioni fra versioni;
- registro `cml-curriculum-version-registry-v1`.

`UNSPECIFIED` e lo scope `repository` servono esclusivamente a rappresentare contenuti presenti nel repository la cui corrispondenza completa a IN 2012 o IN 2025 non è ancora documentata. Non possono essere usati per versioni nazionali o d'istituto.

## Registro iniziale

File:

```text
curman-react/src/data/curriculum-version-registry.json
```

Il registro parte intenzionalmente vuoto:

```json
{
  "schemaVersion": "cml-curriculum-version-registry-v1",
  "generatedAt": null,
  "versions": []
}
```

La registrazione prudente del dataset corrente è rinviata a CML-483.

## Validatore

File:

```text
curman-react/tools/validate-curriculum-version-registry.mjs
```

Comando:

```text
npm run validate:versions
```

Verifica:

- schema version;
- identità univoche e formato `versionId`;
- enum di framework, scope, ordine, lifecycle e deliberazione;
- campi obbligatori e valori nullable;
- anni scolastici consecutivi e intervalli non invertiti;
- regole scope/framework;
- versioni nazionali senza stato deliberativo;
- versioni d'istituto con `institutionId` e stato umano;
- riferimenti fra versioni esistenti e non autoreferenziali;
- assenza di false attribuzioni normative.

## CI

Il comando `npm run test:cml482` è inserito in React CI prima della build TypeScript/Vite.

## Limiti intenzionali

CML-482 non introduce:

- record per il dataset corrente;
- dataset IN 2012 o IN 2025 aggiuntivi;
- selettore di versione;
- modifica a Consultazione, Revisione o Esportazioni;
- modifica ai contenuti curricolari o ai Gap package;
- backend, autenticazione o telemetria;
- deploy manuale.
