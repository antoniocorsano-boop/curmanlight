## Decisione selezionata

**Prossima runtime slice: CML-255 — EXPORTS ROLE-BASED CLARITY MICRO-UX**

Motivazione:
- Dopo aver chiarito percorso (Home), Guida (per ruolo) e messaggi di azione (Processo), la priorità naturale è rendere chiaro il momento finale: **cosa esportare, per quale ruolo e con quale scopo**
- La sezione Esportazioni è funzionale ma priva di collegamento esplicito al flusso di lavoro
- Utenti non esperti faticano a distinguere tra file `.cml`, Word/PDF e copia di sicurezza
- L'intervento è piccolo, a basso rischio, coerentemente con lo stato attuale del progetto

## Perimetro consigliato per CML-255

- Intervento runtime piccolo (microcopy-only in `index.html`)
- Nessuna nuova funzione
- Nessuna modifica a storage
- Nessuna modifica a schema `.cml`
- Nessuna modifica ai contratti import/export
- Nessuna nuova dipendenza
- Nessuna rete/API/OAuth/backend
- Miglioramento microcopy nella sezione Esportazioni
- Eventuale blocco compatto "Quale esportazione usare?"
- Collegamento tra ruolo, file da produrre e momento del percorso
- Nessuna duplicazione pesante della Guida o della Home

## Esempi di logica da valutare

- **Docente:** esporta proposta docente o file di lavoro per il dipartimento
- **Dipartimento:** esporta esito condiviso dopo il confronto
- **Referente:** raccoglie esiti e produce report o documento di sintesi
- **Documento finale:** produce Word/PDF/Markdown o file da conservare
- **Backup:** conserva file di lavoro, senza sostituire il documento finale
