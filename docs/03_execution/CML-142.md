## Opzioni valutate

### A — Adattare adapter a shape flat

Modificare l'adapter per produrre campi `descrizione`, `fonte`, `nodiRiferimento`.

- **Vantaggio**: integrazione diretta
- **Rischio**: adapter meno ricco, perdita informativa
- **Scartata**: distrugge il valore aggiunto dell'adapter

### B — Creare `toRuntimeMappaDati()` (strato intermedio)

Mantenere adapter ricco + secondo trasformatore standalone.

- **Vantaggio**: separa dato ricco da compatibilità runtime, layer testabile
- **Rischio**: un passaggio extra da testare
- **Raccomandata**

### C — Modificare `renderMappaDisciplinare()` per `unita[]`

- **Vantaggio**: usa dati più ricchi
- **Rischio**: viola vincolo "non modificare runtime", regressioni UI
- **Scartata**: vietata dai vincoli

### D — Pilota solo Tecnologia

Prima il ponte solo su Tecnologia, fallback hardcoded per altre.

- **Vantaggio**: rischio controllato
- **Rischio**: doppio regime temporaneo
- **Raccomandata** (come combinazione con B)

### E — Completare tutte le discipline prima

- **Vantaggio**: base dati omogenea
- **Rischio**: mantiene runtime hardcoded ancora più a lungo
- **Scartata**: ritarda senza beneficio immediato

## Opzione selezionata: **B + D**

```
toRuntimeMappaDati(toMappaDati(json)) → shape flat per renderMappaDisciplinare()
```

Prima test su **Tecnologia** come disciplina pilota, mantenendo fallback hardcoded per le altre.
