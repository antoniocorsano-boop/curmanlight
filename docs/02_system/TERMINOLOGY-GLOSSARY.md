# Terminology Glossary

Glossario operativo per mantenere linguaggio coerente e comprensibile per utenti scolastici non tecnici.

## Termini ammessi

- disciplina
- curricolo vigente
- proposta di aggiornamento 2025
- documento
- esporta
- guida
- sezione
- classe
- anno scolastico

## Termini da evitare

- voci (se non definito nel contesto)
- copertura curricolo (nella UI principale)
- runtime
- hash
- workflow
- pacchetto
- shape test
- validator

## Sostituzioni consigliate

| Termine da evitare | Sostituzione consigliata | Nota |
|---|---|---|
| voci | elementi del curricolo / contenuti | Usare "voci" solo se prima definito chiaramente. |
| copertura curricolo | stato del curricolo / avanzamento consultazione | Evitare formulazioni astratte in Home e aree principali. |
| runtime | funzionamento dell'app | Consentito solo in documentazione tecnica interna. |
| hash | codice versione | Non esporre in UI utente. |
| workflow | percorso di lavoro | Preferire linguaggio scolastico-operativo. |
| pacchetto | documento esportato | Terminologia utente finale. |
| shape test | controllo dati interno | Tenere in area tecnica, non in UI. |
| validator | controllo automatico interno | Tenere in area tecnica, non in UI. |

## Regola d'uso

Ogni nuova etichetta UI deve essere verificata contro questo glossario prima della chiusura della slice.
