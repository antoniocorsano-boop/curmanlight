# Protocollo osservazione reale — CurManLight

**Durata consigliata:** 1–2 settimane  
**Gruppo minimo di prova:** DS o delegato, referente curricolo, 1 coordinatore di dipartimento, 2–3 docenti  
**Obiettivo:** Verificare l'usabilità di CurManLight in condizioni reali controllate e raccogliere osservazioni per decidere il passo successivo.

---

## Cosa osservare

Durante la prova, verificare:

1. **Comprensione della home** — all'apertura dell'app, l'utente capisce cosa fare?
2. **Comprensione del viewer curricolo** — l'utente trova le discipline, le versioni e l'indice?
3. **Distinzione IN 2012 / IN 2025** — le etichette e i colori sono chiari?
4. **Fonti (sezioni generali)** — l'utente le raggiunge e le riconosce come quadro d'insieme?
5. **Flusso docente → dipartimento → referente** — il passaggio di file `.cml` è comprensibile?
6. **Utilità del report gruppo curricolo** — il report contiene le informazioni che servono?
7. **Chiarezza delle guide** — le guide sono sufficienti per capire cosa fare?

## Cosa non fare

- Non inserire dati personali (nomi, cognomi, email, dati sensibili) nei file `.cml` o nei registri.
- Non caricare documenti sensibili o protetti da privacy.
- Non usare CurManLight come atto ufficiale o delibera.
- Non considerare la bozza IN 2025 come già approvata.
- Non sostituire con lo strumento le riunioni di dipartimento o le delibere del Collegio Docenti.

## Modalità di raccolta osservazioni

Le osservazioni possono essere raccolte in uno di questi modi:

- **Traccia raccolta osservazioni** (`CML_TRACCIA_RACCOLTA_OSSERVAZIONI.md`) — domande per ruolo.
- **Scheda osservazione per ruolo** (`CML_SCHEDA_OSSERVAZIONE_PER_RUOLO.md`) — scheda strutturata per ciascun profilo.
- **Registro sintetico** (`CML_REGISTRO_SINTETICO_OSSERVAZIONI.md`) — tabella per chi coordina la prova.
- **Liberamente** — ogni partecipante può inviare osservazioni in forma libera.

## Criteri per decidere il passo successivo

Dopo la prova, valutare:

- **Tutti i partecipanti hanno completato le attività minime?**
- **Le difficoltà segnalate sono risolvibili con la documentazione esistente?**
- **Servono modifiche all'app prima di un uso esteso?**
- **Si può passare a una fase operativa con proposte reali su discipline specifiche?**
- **È necessario un nuovo incontro di chiarimento?**

In base alle risposte, si può scegliere tra:

- Aprire un ciclo di proposte reali.
- Migliorare l'interfaccia o la documentazione.
- Estendere gli esempi `.cml` a più discipline.
- Fermarsi e consolidare.
