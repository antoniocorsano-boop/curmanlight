# Checklist avvio pilot osservazioni — CurManLight

## Prima dell'incontro

- [ ] Verificare che l'app sia raggiungibile: [https://antoniocorsano-boop.github.io/curmanlight](https://antoniocorsano-boop.github.io/curmanlight)
- [ ] Preparare i materiali per i partecipanti:
  - [ ] Nota di rilascio
  - [ ] Scheda stato progetto
  - [ ] Guida rapida utente
  - [ ] Protocollo prova controllata
  - [ ] Scheda osservazione per ruolo
  - [ ] Messaggio di invito (se inviato in anticipo)
- [ ] Predisporre un computer con proiettore/schermo per la demo
- [ ] Caricare un file `.cml` di esempio pronto per la dimostrazione
- [ ] Verificare la durata prevista (30–45 min) e confermare l'ordine del giorno
- [ ] Stampare o condividere digitalmente il registro sintetico osservazioni

## Durante l'incontro

- [ ] Aprire l'app live e mostrare le due versioni del curricolo
- [ ] Mostrare le sezioni generali e il callout
- [ ] Mostrare il flusso completo (docente → dipartimento → referente → report)
- [ ] Ribadire che è materiale di lavoro, non atto approvato
- [ ] Ribadire che nessun dato personale viene raccolto
- [ ] Lasciare spazio per domande dopo ogni blocco
- [ ] Distribuire il protocollo prova controllata e la scheda osservazione per ruolo
- [ ] Fissare una data per la riconsegna delle osservazioni

## Subito dopo l'incontro

- [ ] Inviare ai partecipanti i link e i materiali digitali
- [ ] Verificare che tutti i partecipanti abbiano ricevuto i materiali
- [ ] Appuntare eventuali domande emerse durante l'incontro
- [ ] Confermare la data di riconsegna delle osservazioni

## Durante la settimana di prova

- [ ] Restare disponibile per chiarimenti via email o messaggio
- [ ] Non sollecitare più di un promemoria a metà periodo
- [ ] Non raccogliere dati personali o sensibili nelle comunicazioni
- [ ] Registrare eventuali richieste di assistenza nel registro sintetico

## Chi raccoglie le osservazioni

Le osservazioni possono essere raccolte da:
- Referente curricolo
- Conduttore dell'incontro
- Un docente incaricato dal DS

## Dove registrarle

- Nel registro sintetico (`CML_REGISTRO_SINTETICO_OSSERVAZIONI.md`)
- Oppure in un file separato con la stessa struttura

## Cosa evitare

- Non modificare i file `.cml` degli esempi
- Non usare nomi reali di persone o studenti
- Non conservare osservazioni in formato che contiene dati personali
- Non presentare le osservazioni come approvazione del curricolo

## Criteri minimi per chiudere la prova

- Almeno un partecipante per ruolo ha completato le attività minime
- Le osservazioni raccolte sono sufficienti per una decisione
- Non ci sono blocchi tecnici che impediscono l'uso
- Il gruppo di prova conferma di aver capito lo scopo dello strumento
