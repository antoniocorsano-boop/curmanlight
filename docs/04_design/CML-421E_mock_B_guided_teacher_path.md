# CML-421E — Mock B: Percorso guidato per docente

## 1. Idea generale

Mock pensato per docenti non tecnici che devono capire subito cosa fare.

Priorità:

```text
orientamento operativo
linguaggio semplice
azioni guidate
pochi elementi per schermata
```

## 3. Flusso principale

```text
Consulta → Progetta → Esporta → Verifica
```

## 4. Curricolo

Vista guidata:

```text
1. Scegli disciplina
2. Scegli ordine/classe
3. Scegli versione
4. Leggi contenuto
5. Usa in progettazione o esporta
```

Avvisi semplici:

```text
IN2025 è materiale/proposta da verificare.
La validazione resta umana.
```

## 5. Progettazione didattica

Vista guidata:

```text
Che cosa vuoi preparare?

[Programmazione annuale]
[UDA]
[Evidenze per attività]
[Collegamenti al curricolo]
```

Programmazione:

```text
1. Classe
2. Periodo
3. Unità
4. Obiettivi
5. Metodologie
6. Verifiche
7. Esporta bozza
```

UDA:

```text
1. Titolo
2. Classe
3. Evidenze
4. Attività
5. Valutazione
6. Esporta bozza
```

## 6. Esportazione

Vista guidata:

```text
Che documento devi preparare?

Per la classe:
[Programmazione annuale]
[Programma svolto]
[Relazione finale]
[Sintesi percorso]

Per il processo:
[Proposta docente .cml]
[Esito dipartimentale]
[Report referente]
```

Se il contesto manca:

```text
Completa prima il Contesto di lavoro nelle Impostazioni.
```

## 7. Riferimenti e guida

```text
Fonti e normativa
IN2012 e IN2025
Ruoli e processo
Uso e limiti
```

Ogni sezione usa frasi brevi e casi pratici.

## 8. Impostazioni / Contesto di lavoro

Wizard breve:

```text
1. Chi sei?
2. Dove lavori?
3. Quale disciplina/ambito?
4. Quali classi?
5. Quale anno scolastico?
```

## 10. Punti forti

```text
molto chiaro per docente
adatto a test utente non tecnico
riduce sovraccarico

```

## 11. Rischi

```text
meno potente per referente/dipartimento
può nascondere dettagli di fonti/versioni
rischia di sembrare troppo procedurale
```
