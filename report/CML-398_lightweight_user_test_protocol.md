# CML-398 — Protocollo di test utente leggero

## Sintesi
Protocollo leggero per validazione con utenti focalizzato sul percorso operativo: consulta → scegli → progetta → esporta → verifica.

## Obiettivi della validazione
1. Verificare la comprensione del percorso operativo
2. Identificare difficoltà di navigazione e comprensione
3. Valutare l'utilità del sistema per i ruoli target
4. Raccogliere feedback per miglioramenti futuri

## Profilo minimo dei partecipanti
- **Docente:** 1-2 partecipanti (scuola secondaria I grado)
- **Coordinatore di dipartimento:** 1 partecipante
- **Referente curricolo:** 1 partecipante
- **Totale:** 3-4 partecipanti

## Scenari di test

### Docente
**Compito:** Progettare attività didattica su Italiano per classe seconda
**Passi:** Consulta → Scegli → Progetta → Esporta → Verifica
**Durata:** 10-15 minuti

### Dipartimento
**Compito:** Verificare coerenza scelte disciplinari
**Passi:** Consulta → Scegli → Progetta → Esporta → Verifica
**Durata:** 10-15 minuti

### Referente
**Compito:** Supervisionare processo complessivo e preparare report
**Passi:** Consulta → Scegli → Progetta → Esporta → Verifica
**Durata:** 10-15 minuti

## Griglia osservativa

### Comprensione del percorso
- Immediatezza nell'iniziare
- Chiarezza del percorso operativo
- Completamento senza esitazioni

### Usabilità
- Facilità nel trovare le sezioni
- Intuitività dell'interfaccia
- Assenza di confusione

### Utilità
- Utilità del materiale esportato
- Supporto nello svolgimento del compito
- Intenzione di utilizzo nella pratica

## Criteri di successo

### Successo pieno
- Completamento senza assistenza
- Comprensione immediata del percorso
- Materiale esportato utile e coerente
- Soddisfazione espressa

### Successo parziale
- Completamento con minima assistenza
- Comprensione dopo spiegazione
- Materiale parzialmente utile
- Atteggiamento neutrale

### Insuccesso
- Incompletamento dei passi
- Mancata comprensione del percorso
- Materiale non utile
- Insoddisfazione espressa

## Criteri di criticità

### Bloccante
- Impossibilità di iniziare
- Errori tecnici del sistema
- Materiale esportato errato
- Abbandono del test

### Gestibile
- Difficoltà in uno o più passi
- Necessità di spiegazioni
- Materiale incompleto
- Suggerimenti significativi

## Privacy e dati

### NON raccogliere
- Dati personali
- Informazioni identificative
- Registrazioni audio/video
- Screenshot con dati sensibili

### Raccogliere
- Osservazioni qualitative
- Feedback su usabilità
- Suggerimenti di miglioramento
- Tempo di completamento (opzionale)
- Livello di soddisfazione (1-5)
- Problemi riscontrati

## Catena successiva

### CML-399 — User validation scenario pack
Esecuzione del test con 1-2 utenti per ruolo e raccolta osservazioni.

### CML-400 — User validation readiness checkpoint
Analisi risultati e decisione sulla prossima direzione.
