# CML-391 — Progetta terminology targeted microfix

## Sintesi
Correzione mirata della terminologia: "Compila" → "Progetta" solo dove si riferisce alla sezione di progettazione didattica, mantenendo "compila" nei dati curricolari e nelle istruzioni form-filling.

## Cambiamenti principali

### UI progettazione didattica (MODIFICATO)
- Tab label desktop e mobile: "Compila" → "Progetta"
- Section headers: "Compila — Evidenze e criteri" → "Progetta — Evidenze e criteri" (e simili)
- Home card: "Compila" → "Progetta"
- Percorso operativo: "compila la proposta" → "progetta la proposta"
- Istruzioni di progettazione: "Compila una proposta didattica" → "Progetta una proposta didattica"
- Esportazioni: "compila nelle aree operative" → "progetta nelle aree operative"
- Guida: "Preparare o compilare una proposta" → "Progettare una proposta"
- Onboarding: "Prepara la proposta in Compila" → "Progetta la proposta"
- JavaScript label maps: tutti i riferimenti a "Compila" → "Progetta"

### Dati curricolari e parole composte (NON MODIFICATO)
- "L'alunno compila una tabella..." (obiettivo curricolare)
- "Compilare una scheda di autovalutazione" (obiettivo curricolare)
- "precompilato" / "precompilati" (template pre-filled)
- "compila i campi e premi" (form-filling instruction)

## Risultato
- Lessico professionale: "progettare" invece di "compilare" per l'azione didattica
- Percorso operativo: consulta → scegli → progetta → esporta → verifica
- Coerenza cross-piattaforma desktop/mobile
- Nessun impatto su dati curricolari o istruzioni form-filling
