# Hermes Assisted Mode Governance Contract

## Purpose
Define the operational contract for using Hermes in assisted mode on the CurManLight project, ensuring safe, auditable, and controlled use of Git operations.

## Operational Rules
1. **No Automatic Pushes**
   - Hermes must not execute `git push`. Any future push must be performed manually by the user or through a separate explicitly authorized controlled-push slice.
   - All commits remain local. Any future push must be performed manually by the user or through a separate explicitly authorized controlled-push slice.

2. **Pre‑Commit Disclosure**
   Before any commit is attempted, Hermes must display the following information:
   - `git status --short --branch`
   - List of modified files
   - Synthetic diff summary
   - Proposed commit message

3. **Docs‑Only Slice Restrictions**
   - Only slices targeting the following paths are classified as docs‑only:
     - `docs/03_execution/`
     - `report/`
     - `docs/REPO-MOVELOG.md`
   - Any slice modifying other paths is considered runtime or another category and requires explicit approval.

4. **Runtime and Deploy Slices**
   - Runtime slices must be dedicated (single-purpose) and require explicit user approval before execution.
   - Deploy slices are a subset of runtime slices that affect production artifacts; they require separate explicit approval.

5. **README.md Prohibition**
   - Modifications to `README.md` are forbidden unless an explicit request is made by the user.

6. **Secret Scan**
   - A minimal secret scan is performed on each operation; detection of any credential or secret aborts the operation immediately.

7. **Verdict and Local Retention**
   - The current verdict `CML_242_HERMES_ASSISTED_MODE_GOVERNANCE_CONTRACT_READY_LOCAL_NOT_PUSHED` indicates that the commit should remain local pending manual push authorization.
